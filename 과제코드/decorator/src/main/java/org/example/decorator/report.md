# A-3: 추상화
~~~ 
체크포인트: "구현체를 바꿨는데 클라이언트가 안 바뀐다" — 이게 추상화의 효과다. 
왜 가능한지 한 문장으로 설명을 주석에 적어라.
~~~
`NotificationService`가 구현체가 아니라 인터페이스 `NotificationSender`에 의존하게 만들었다.
그로 인해 동일한 인터페이스를 구현한 객체라면 생성자를 통해 교체할 수 있다.
따라서 구현체가 변경되어도 `NotificationService`의 코드는 변경되지 않는다.
