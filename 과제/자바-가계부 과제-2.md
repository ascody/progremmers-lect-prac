# 가계부 만들기 2 (File I/O 활용)

> 이번에는 데이터를 메모리(컬렉션)가 아니라 **날짜별 텍스트 파일(`.txt`)** 에 저장합니다. 프로그램을 껐다 켜도 기록이 남는다는 점이 1편과의 가장 큰 차이입니다. **아래 Step을 순서대로 따라가면, 마지막 Step에서 프로그램이 완성됩니다.**

---

## 1. 무엇을 만드나요?

콘솔에서 동작하는 가계부입니다. 단, 모든 내역은 **파일로 저장**됩니다.

- 하루치 기록 = 파일 1개 (예: `2024-09-04.txt`)
- 내역을 추가하면 그 날짜 파일에 글이 쌓이고,
- 조회하면 파일을 읽어서 보여주고,
- 삭제하면 그 파일을 통째로 지웁니다.

실행 화면 메뉴:

```
===== 가계부 (File) =====
1. 내역 추가
2. 내역 조회
3. 삭제
4. 종료
번호 입력 >
```

## 2. 요구사항 정리 (기능 명세)

|번호|기능|설명|
|---|---|---|
|1|내역 추가|**오늘 날짜**의 파일이 없으면 새로 만들고, 있으면 기존 파일 **뒤에 이어서** 내용을 추가한다.|
|2|내역 조회|저장된 파일들의 **날짜 목록**을 보여준다. 날짜를 입력하면 그 파일의 내용을 읽어 출력한다.|
|3|삭제|날짜를 입력받아 **그 날짜의 파일을 삭제**한다.|
|4|종료|프로그램을 끝낸다.|

### 파일에 저장되는 형식

```
공책 : 1000원
연필 : 300원
합계 : 1300원
```

## 3. 핵심 개념: 파일을 어떻게 다룰까?

1편은 `Map`에 저장했지만, 이번엔 **파일이 곧 데이터**입니다.

|하고 싶은 것|사용하는 도구|
|---|---|
|오늘 날짜 구하기|`LocalDate.now()` → 문자열 `"2024-09-04"`|
|파일이 있는지 확인|`File.exists()`|
|파일에 **이어서** 쓰기(append)|`new FileWriter(파일, true)` ← `true`가 핵심|
|파일 읽기|`BufferedReader.readLine()` 반복|
|폴더 안 파일 목록|`폴더.list()` 또는 `폴더.listFiles()`|
|파일 삭제|`File.delete()`|

> **중요**: 파일을 다루는 코드는 **반드시 예외 처리(`try-catch` 또는 `throws IOException`)** 가 필요합니다. 파일이 없거나 권한 문제가 생길 수 있기 때문입니다.

### 저장 위치 정하기

파일이 여기저기 흩어지지 않도록, 프로젝트 안에 **`accountbook`** 폴더 하나를 정해서 그 안에만 `.txt`를 저장하는 것을 추천합니다.

```
프로젝트/
 └─ accountbook/
     ├─ 2024-09-04.txt
     ├─ 2024-09-03.txt
     └─ ...
```

## 4. 파일 구조 (각 파일의 역할)

|파일|역할|
|---|---|
|`AccountBook.java` _(인터페이스)_|기능 3개(추가/조회/삭제)를 **선언**만 한다|
|`AccountBookImpl.java`|`AccountBook`을 **구현**한다. 실제 File I/O 로직이 들어간다|
|`Start.java`|`main` 메서드. 메뉴 출력 + 입력에 따라 기능 호출|

> 1편과 달리 `Item` 같은 데이터 클래스는 꼭 필요하진 않습니다. 항목들을 파일에 글로 쓰면 되니까요. (단, 추가할 때 합계를 계산하려면 입력값을 잠깐 모아두는 정도는 필요합니다.)

### Step 1. 메뉴부터 띄우기 (`Start.java`)

**목표**: 기능이 없어도 메뉴가 반복 출력되고 `4`를 누르면 종료되게 만든다.

**할 일**

- `Scanner`로 번호 입력
- `while` 반복문 + `switch`로 메뉴 분기
- `4`면 반복 종료

**힌트**

```java
Scanner sc = new Scanner(System.in);
while (true) {
    System.out.println("===== 가계부 (File) =====");
    // 1~4 메뉴 출력
    int menu = Integer.parseInt(sc.nextLine());
    switch (menu) {
        case 1: break; // 나중에
        case 2: break;
        case 3: break;
        case 4: System.out.println("종료합니다"); return;
        default: System.out.println("잘못된 번호입니다");
    }
}
```

**확인**: 4를 누르면 종료되면 성공.

### Step 4. 기능 1 — 내역 추가 (`addAccount`)

**목표**: 오늘 날짜 파일에 항목들을 이어서 저장하고, 합계를 함께 기록한다.

**할 일**

1. 오늘 날짜로 파일 경로를 만든다. (예: `accountbook/2024-09-04.txt`)
2. 항목 이름·금액을 반복 입력받아 임시로 모으고 합계를 계산한다.
3. **append 모드**(`new FileWriter(file, true)`)로 파일에 형식대로 쓴다.
    - 파일이 없으면 `FileWriter`가 알아서 만들어 준다. (별도 `createNewFile` 불필요)
4. 저장한 내용을 화면에도 출력한다.

**힌트**

```java
String today = LocalDate.now().toString();      // "2024-09-04"
File file = new File(DIR, today + ".txt");

int total = 0;
StringBuilder sb = new StringBuilder();
// 반복: 이름/금액 입력 → sb에 "이름 : 금액원\n" 추가, total += 금액
// 반복 끝나면 sb에 "합계 : total원\n" 추가

try (FileWriter fw = new FileWriter(file, true)) { // true = 이어쓰기
    fw.write(sb.toString());
} catch (IOException e) {
    System.out.println("저장 중 오류: " + e.getMessage());
}
```

**확인**: `accountbook` 폴더에 오늘 날짜 `.txt`가 생기고, 열어보면 형식대로 들어 있으면 성공. 한 번 더 추가하면 같은 파일 **뒤에** 이어 붙으면 완벽.

> **설계 결정 포인트**: 하루에 여러 번 추가하면 `합계` 줄도 여러 번 생깁니다. 이대로 둬도 되고(그때그때의 합계), 더 깔끔하게 하려면 "기존 파일을 읽어 전체 합계를 다시 계산해 마지막 줄을 갱신"하도록 발전시킬 수 있습니다.

### Step 5. 기능 2 — 내역 조회 (`showAccount`)

**목표**: 저장된 날짜 목록을 보여주고, 선택한 날짜의 파일 내용을 읽어 출력한다.

**할 일**

1. 저장 폴더의 `.txt` 파일 목록을 가져온다.
2. 파일이 하나도 없으면 "기록이 없습니다" 출력 후 종료.
3. 파일명에서 `.txt`를 떼고 날짜만 출력한다.
4. 날짜를 입력받아 해당 파일이 있는지 확인.
    - 없으면 "그런 날짜가 없습니다".
    - 있으면 `BufferedReader`로 한 줄씩 읽어 출력.

**힌트**

```java
File folder = new File(DIR);
String[] files = folder.list();           // 파일명 배열
// files 가 null 이거나 length==0 이면 기록 없음

for (String name : files) {
    if (name.endsWith(".txt"))
        System.out.println(name.replace(".txt", "")); // 날짜만
}

// 날짜 입력 후 읽기
File file = new File(DIR, 입력날짜 + ".txt");
if (file.exists()) {
    try (BufferedReader br = new BufferedReader(new FileReader(file))) {
        String line;
        while ((line = br.readLine()) != null) {
            System.out.println(line);
        }
    } catch (IOException e) { /* 오류 처리 */ }
}
```

**확인**: 날짜 목록이 나오고, 날짜를 입력하면 그 파일 내용이 그대로 출력되면 성공.

---

### Step 6. 기능 3 — 삭제 (`deleteAccount`)

**목표**: 입력한 날짜의 파일을 삭제한다.

**할 일**

1. (편의상) 현재 날짜 목록을 먼저 보여준다.
2. 삭제할 날짜를 입력받는다.
3. 해당 파일이 있는지 확인 후 `delete()` 한다.
4. 결과(성공/없음)를 안내한다.

**힌트**

```java
File file = new File(DIR, 입력날짜 + ".txt");
if (file.exists()) {
    if (file.delete()) System.out.println("삭제되었습니다");
    else               System.out.println("삭제 실패");
} else {
    System.out.println("그런 날짜가 없습니다");
}
```

**확인**: 삭제 후 "내역 조회"하면 그 날짜가 목록에서 사라져 있으면 성공.

## 7. (선택) 도전 과제

1. **전체 합계 갱신**: 같은 날 여러 번 추가해도 합계 줄이 하나로 유지되도록 (읽기→재계산→다시 쓰기)
2. **정렬**: 조회 시 날짜 목록을 최신순으로 정렬해서 출력
3. **검색**: 특정 항목 이름(예: "공책")이 들어간 날짜를 모두 찾아 출력
4. **백업**: 삭제 전에 파일을 `backup` 폴더로 옮긴 뒤 지우기
5. **월별 합계**: 같은 달(예: 2024-09)의 모든 파일 금액을 합쳐서 출력
