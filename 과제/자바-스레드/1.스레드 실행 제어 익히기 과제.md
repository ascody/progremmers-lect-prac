# 스레드 실행 제어 익히기 (sleep · interrupt · yield · join)

> 스레드의 **상태(state)** 와 **실행을 제어하는 메서드**들을 직접 실험하며 익힙니다.
> 각 Step은 작은 실험 하나예요. 코드를 돌려보고 **무슨 일이 일어나는지 관찰**하는 게 핵심입니다.
> **아래 Step을 순서대로 따라가면, 스레드 제어의 핵심 메서드를 모두 경험하게 됩니다.**
>
> 💡 각 Step의 **힌트는 접혀 있어요.** 먼저 스스로 해보고, 막히면 "힌트 보기"를 펼치세요.

---

## 0. 먼저 알아둘 점

스레드 실행 순서는 **운영체제와 스케줄러가 정하기 때문에 비결정적(non-deterministic)** 이에요. 그래서 같은 코드라도 **실행할 때마다 출력 순서가 다를 수 있습니다.** "내 결과가 친구와 다르다"가 정상이에요. 우리는 정확한 순서가 아니라 **"각 메서드가 어떤 효과를 내는가"** 를 관찰하는 게 목표입니다.

---

## 1. 무엇을 하나요?

스레드를 만들어 4가지 제어 메서드를 하나씩 실험합니다.

```
실험1: sleep()     → 잠재우기 (그리고 흔한 함정 하나)
실험2: interrupt() → 멈춰달라 "요청"하기 (busy 스레드)
실험3: interrupt() → 자고 있는 스레드 깨우기 (InterruptedException)
실험4: yield()     → CPU 양보(힌트)
실험5: join()      → 다른 스레드가 끝나길 기다리기
```

---

## 2. 학습 목표

| 개념 | 어디서 배우나 |
|------|------|
| 스레드 상태 | NEW / RUNNABLE / BLOCKED / WAITING / TERMINATED |
| `sleep()` | 현재 스레드를 일정 시간 멈춤 (static!) |
| `interrupt()` | 작업 취소 "요청" + 일시정지 스레드 깨우기 |
| `yield()` | 실행 시간을 다른 스레드에 양보(힌트) |
| `join()` | 다른 스레드가 끝날 때까지 기다림 |

---

## 3. 핵심 개념

### (1) 스레드의 5가지 상태
```
NEW ──start()──▶ RUNNABLE ◀──▶ (실행)
                   │  ▲
                   │  └── BLOCKED      (락 대기)
                   │  └── WAITING / TIMED_WAITING  (sleep/wait/join)
                   ▼
               TERMINATED  (작업 종료)
```
- **NEW**: 생성됐지만 `start()` 전
- **RUNNABLE**: 실행 중이거나 실행 대기열에서 차례를 기다림
- **BLOCKED**: 동기화 락(lock)이 풀리길 기다림
- **WAITING / TIMED_WAITING**: `sleep()`·`wait()`·`join()` 등으로 일시정지 (TIMED는 시간 지정)
- **TERMINATED**: 작업이 끝나 소멸

### (2) 제어 메서드와 상태 변화
| 메서드 | 효과 | 비고 |
|--------|------|------|
| `sleep(ms)` | 현재 스레드를 ms동안 멈춤 → TIMED_WAITING | **static** (현재 실행 중인 스레드에 작동) |
| `join()` | 호출한 스레드가, 대상이 끝날 때까지 기다림 | 대상이 끝나면 다시 진행 |
| `interrupt()` | 멈추라고 "요청". 일시정지 중이면 깨움 | 강제 종료가 아님! |
| `yield()` | 남은 실행 시간을 양보(힌트) | 보장은 없음 |

> `stop()`·`suspend()`·`resume()`은 교착 상태를 만들기 쉬워 **deprecated**(쓰지 말 것)예요.

### (3) interrupt 3형제 구분 (헷갈림 주의)
- `interrupt()` — 대상 스레드에게 "멈춰달라" 요청 (인터럽트 플래그를 켬)
- `isInterrupted()` — (인스턴스) 인터럽트가 요청됐는지 확인. **플래그 유지**
- `interrupted()` — (static) 현재 스레드 확인 후 **플래그를 초기화**(끔)

## 5. Step by Step

### Step 1. 준비 — 스레드 두 개 동시 실행

**목표**: `Thread`를 상속해 `run()`을 만들고 `start()`로 실행하는 기본을 복습한다.

**할 일**
1. `-`를 300번 출력하는 스레드, `|`를 300번 출력하는 스레드.
2. `main`에서 둘 다 `start()`.

<details>
<summary>💡 힌트 보기</summary>

```java
class PrintDash extends Thread {
    public void run() {
        for (int i = 0; i < 300; i++) System.out.print("-");
    }
}
class PrintBar extends Thread {
    public void run() {
        for (int i = 0; i < 300; i++) System.out.print("|");
    }
}
// main에서:
new PrintDash().start();
new PrintBar().start();
```

`run()`을 직접 부르면 안 돼요! `start()`를 호출해야 새 스레드로 실행됩니다.

</details>

**관찰 포인트**: `-`와 `|`가 **뒤섞여** 나오면 두 스레드가 동시에 도는 거예요. 실행할 때마다 섞이는 모양이 달라집니다.

---

### Step 2. sleep() — 그리고 흔한 함정

**목표**: `sleep()`으로 스레드를 멈춰보고, **`sleep()`이 static이라는 함정**을 관찰한다.

**할 일**
1. 스레드 `run()` 안에서 작업 후 `Thread.sleep(2000)` 하고 "종료" 출력.
2. (함정 실험) `main`에서 `t1.sleep(2000)`을 호출해 본다.

<details>
<summary>💡 힌트 보기</summary>

```java
class SleepThread extends Thread {
    public void run() {
        for (int i = 0; i < 300; i++) System.out.print("-");
        try {
            Thread.sleep(2000);     // 이 스레드를 2초 멈춤
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
        System.out.println("<<종료>>");
    }
}
```

**함정**: `main`에서 `t1.sleep(2000)`이라고 써도, 실제로 잠드는 건 t1이 아니라 **그 코드를 실행 중인 main 스레드**예요. `sleep()`은 static이라 "현재 실행 중인 스레드"에 작동하거든요. 그래서 항상 `Thread.sleep(...)` 형태로 쓰는 게 맞아요.

</details>

**관찰 포인트**: `t1.sleep()`을 써도 t1이 멈추지 않는다(=main이 멈춘다). "참조변수.sleep()"이 왜 헷갈리는 코드인지 체감하기.

---

### Step 3. interrupt() — 멈춰달라 "요청"하기

**목표**: `interrupt()`는 강제 종료가 아니라 **요청**일 뿐이고, 스레드가 `isInterrupted()`로 직접 확인해야 멈춘다는 걸 본다.

**할 일**
1. `while (i != 0 && !isInterrupted())`로 카운트다운(중간에 긴 빈 반복으로 시간 지연).
2. `main`에서 입력을 받은 뒤 `t1.interrupt()` 호출.

<details>
<summary>💡 힌트 보기</summary>

```java
class CountThread extends Thread {
    public void run() {
        int i = 10;
        while (i != 0 && !isInterrupted()) {   // 인터럽트 요청 오면 멈춤
            System.out.println(i--);
            for (long x = 0; x < 2_500_000_000L; x++) ;  // 시간 지연(busy)
        }
        System.out.println("카운트가 종료되었습니다.");
    }
}
// main:
CountThread t1 = new CountThread();
t1.start();
new Scanner(System.in).nextLine();   // 아무 값 입력 대기
t1.interrupt();                       // 멈춰달라 요청
```

`interrupt()`는 "플래그만 켜는" 요청이에요. 스레드가 `isInterrupted()`로 그 플래그를 확인하지 않으면 계속 돕니다.

</details>

**관찰 포인트**: 입력하기 전엔 카운트가 계속 돌고, **입력(=interrupt) 후 카운트가 멈추는지** 확인. `interrupt()` 전후로 `isInterrupted()` 값이 `false → true`로 바뀝니다.

---

### Step 4. interrupt() — 자고 있는 스레드 깨우기

**목표**: `sleep()`·`wait()`·`join()`으로 **일시정지 중인** 스레드를 `interrupt()`하면 **InterruptedException**이 터지며 깨어난다는 걸 본다.

**할 일**: Step 3의 카운트다운에서 시간 지연을 `Thread.sleep(2000)`으로 바꾸고, `main`에서 `interrupt()`.

<details>
<summary>💡 힌트 보기</summary>

```java
class CountSleepThread extends Thread {
    public void run() {
        int i = 10;
        while (i != 0 && !isInterrupted()) {
            System.out.println(i--);
            try {
                Thread.sleep(2000);   // 자고 있는 동안 interrupt가 오면?
            } catch (InterruptedException e) {
                System.out.println("자다가 깨어남! (InterruptedException)");
                break;   // 깨어나면 반복 종료
            }
        }
        System.out.println("카운트가 종료되었습니다.");
    }
}
```

자고 있을 때 `interrupt()`가 오면 `sleep()`이 **즉시** `InterruptedException`을 던지며 깨어나요. (참고: 이 예외가 발생하면 인터럽트 플래그는 다시 꺼집니다.)

</details>

**관찰 포인트**: Step 3에선 현재 반복이 끝나야 멈췄지만, 여기선 자고 있다가 **즉시** 예외로 깨어나는 차이를 관찰.

---

### Step 5. yield() — CPU 양보(힌트)

**목표**: `yield()`가 "다른 스레드에게 양보" 신호일 뿐, **보장은 없다**는 걸 관찰한다.

**할 일**: 이름을 가진 스레드 둘이 5번 반복 출력하며 매 반복마다 `Thread.yield()`.

<details>
<summary>💡 힌트 보기</summary>

```java
class YieldThread extends Thread {
    private String name;
    public YieldThread(String name) { this.name = name; }
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println(name + " 실행 중. 반복: " + i);
            Thread.yield();   // 남은 실행시간 양보(힌트)
            try { Thread.sleep(500); } catch (InterruptedException e) { break; }
        }
    }
}
// main: new YieldThread("스레드1").start(); new YieldThread("스레드2").start();
```

`yield()`는 스케줄러에게 보내는 **힌트**일 뿐이에요. 실제로 양보될지는 OS·스케줄러가 정합니다.

</details>

**관찰 포인트**: 두 스레드가 번갈아 나오는 경향이 있지만 **항상 그렇진 않다.** 여러 번 실행해 순서가 달라지는지 보기.

---

### Step 6. join() — 다른 스레드가 끝나길 기다리기

**목표**: `join()`으로 main이 다른 스레드들이 끝날 때까지 **기다리게** 한다.

**할 일**
1. 오래 도는 스레드 둘을 `start()`.
2. `main`에서 `t1.join(); t2.join();` 후 소요시간 출력.

<details>
<summary>💡 힌트 보기</summary>

```java
ManyPrintThread t1 = new ManyPrintThread('-');
ManyPrintThread t2 = new ManyPrintThread('|');
t1.start();
t2.start();

long start = System.currentTimeMillis();
try {
    t1.join();   // t1이 끝날 때까지 main이 기다림
    t2.join();   // t2도
} catch (InterruptedException e) {
    throw new RuntimeException(e);
}
System.out.println("소요시간: " + (System.currentTimeMillis() - start) + "ms");
```

`join()`이 없으면 main은 두 스레드를 켜두고 **먼저 끝나버려요**. `join()` 덕분에 둘 다 마칠 때까지 기다립니다.

</details>

**관찰 포인트**: `join()`을 **뺐을 때**(main이 먼저 끝남)와 **넣었을 때**(둘 다 끝난 뒤 소요시간 출력)를 비교.

---

### Step 7. 마무리 — 상태 전이로 정리 (완성!)

**점검 항목**
- [ ] `start()`가 NEW → RUNNABLE 전이임을 안다
- [ ] `sleep()`이 static이라 "현재 스레드"에 작동하는 함정을 설명할 수 있다
- [ ] `interrupt()`가 강제 종료가 아니라 "요청"임을 안다
- [ ] 일시정지(sleep 등) 중 interrupt → InterruptedException을 관찰했다
- [ ] `yield()`는 보장 없는 힌트임을 안다
- [ ] `join()`이 "끝날 때까지 기다리기"임을 안다

여기까지 통과하면 **스레드 제어 실험 완료입니다!** 🎉

## 8. (선택) 도전 과제

1. **상태 출력**: `t1.getState()`로 NEW/RUNNABLE/TIMED_WAITING/TERMINATED가 실제로 바뀌는지 출력해 보기
2. **interrupted() 비교**: `isInterrupted()`(플래그 유지)와 `interrupted()`(플래그 초기화)의 차이를 실험으로 확인
3. **join(시간)**: `join(1000)`처럼 시간 지정 join을 써서 "최대 1초만 기다리기" 구현
4. **데몬 스레드**: `setDaemon(true)` 스레드는 main이 끝나면 같이 종료된다 — 실험으로 확인
5. **올바른 예외 처리**: `catch (InterruptedException)`에서 `RuntimeException`을 던지는 대신, 인터럽트 상태를 복원(`Thread.currentThread().interrupt()`)하는 더 안전한 방식 알아보기
6. **Runnable 방식**: `Thread` 상속 대신 `Runnable` 구현으로 같은 실험을 다시 해보기 (실무에서 더 권장되는 방식)
