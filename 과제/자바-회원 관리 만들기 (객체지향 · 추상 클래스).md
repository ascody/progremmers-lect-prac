# 회원 관리 만들기 (객체지향 · 추상 클래스)

> 2차원 배열(`String[][]`)로 짠 회원 관리 프로그램을 **객체지향 + 추상 클래스**로 다시 설계합니다. 핵심 전환은 **`String[][]` → `Member` 객체 배열**이에요. 그리고 추상 클래스가 제 역할을 하도록 **회원 등급(일반/VIP)** 을 추가해, 등급마다 혜택이 다르게 만듭니다. **아래 Step을 순서대로 따라가면, 마지막 Step에서 프로그램이 완성됩니다.**

---

## 1. 무엇을 만드나요?

요금제로 정원을 정하고, 회원을 추가·조회·수정·삭제합니다. 회원은 **일반/VIP 등급**을 가지며 등급마다 혜택이 다릅니다.

```
[수행할 업무 - 현재 회원수 : 1/20]
[1]회원추가 [2]회원조회(메일) [3]회원조회(이름)
[4]전체조회 [5]수정 [6]삭제 [7]종료
> 4
[VIP] 홍길동 / hong@a.com / 010-1111 (혜택: 10% 할인 + 무료배송)
[일반] 김철수 / kim@b.com / 010-2222 (혜택: 기본 서비스)
```

> 기능은 원래 버전과 같습니다. 달라지는 건 **구조** — 흩어진 `members[i][0/1/2]` 인덱싱이 깔끔한 객체로 바뀝니다.

---

## 2. 학습 목표

|개념|어디서 배우나|
|---|---|
|클래스로 데이터 묶기|`String[][]` 대신 `Member` 객체로 회원 표현|
|추상 클래스 / 추상 메서드|회원 공통 틀 `Member`, 등급별로 강제되는 `getBenefit()`|
|상속 (`extends`, `super`)|`NormalMember`/`VipMember`가 `Member`를 물려받음|
|다형성|`Member[]`에 일반·VIP를 섞어 담고 똑같이 다룬다|
|캡슐화|회원 데이터·CRUD 로직을 클래스 안으로 숨긴다|

---

## 3. 핵심 개념

### (1) ⭐ 2차원 배열 → 객체

원래는 회원 한 명을 `members[i][0]`(이름), `members[i][1]`(이메일)처럼 **인덱스 번호**로 다뤘어요. 이건 헷갈리고 실수하기 쉽습니다. 이제는 회원을 **객체**로 만듭니다.

```
String[][] members      →   Member[] members
members[i][1]           →   members[i].getEmail()
```

"1번 열이 이메일이었나 연락처였나?" 고민할 필요가 없어집니다.

### (2) 추상 클래스 Member + 등급 자식

모든 회원은 이름·이메일·연락처를 공통으로 가집니다 → 부모 `Member`에 둠. 하지만 **등급별 혜택**은 다릅니다 → 추상 메서드로 강제하고 자식이 구현.

```java
public abstract String getBenefit();   // 일반/VIP가 각자 다르게 구현
```

### (3) 다형성

```java
Member[] members = new Member[정원];
members[0] = new VipMember(...);
members[1] = new NormalMember(...);
members[0].printInfo();   // 각 회원의 등급·혜택이 알아서 다르게 출력됨
```

### (4) 캡슐화 — MemberManager

원래 `main`과 static 메서드에 흩어져 있던 회원 배열·인원수·CRUD 로직을 `MemberManager` 클래스 하나로 모읍니다.

---

## 4. 파일 구조

|파일|역할|
|---|---|
|`Member.java`|**추상 클래스**. 회원 공통(이름·이메일·연락처) + 추상 메서드(`getGrade`, `getBenefit`)|
|`NormalMember.java` / `VipMember.java`|`Member`를 상속한 등급별 회원|
|`MemberManager.java`|회원 배열과 CRUD(추가/조회/수정/삭제) 관리|
|`Main.java`|요금제 선택 + 메뉴 루프|

## 5. Step by Step

각 Step에 **목표 / 할 일 / 힌트 / 확인 방법**이 있습니다.

---

### Step 1. 회원 공통 틀 — 추상 클래스 (`Member.java`)

**목표**: 모든 회원이 공유하는 정보와, 등급별로 강제할 메서드를 가진 추상 클래스를 만든다.

**할 일**

1. `abstract class Member`.
2. 필드 `name`, `email`, `phone` (`protected`) + 생성자 + getter.
3. 추상 메서드 `getGrade()`(등급 이름), `getBenefit()`(혜택).
4. 공통 출력 `printInfo()` — 추상 메서드를 활용해 한 줄로 출력.

**힌트**

```java
public abstract class Member {
    protected String name, email, phone;

    public Member(String name, String email, String phone) {
        this.name = name;
        this.email = email;
        this.phone = phone;
    }

    public String getName()  { return name; }
    public String getEmail() { return email; }
    public String getPhone() { return phone; }

    public abstract String getGrade();    // 자식이 구현
    public abstract String getBenefit();  // 자식이 구현

    // 추상 메서드를 부모의 구현 메서드가 활용한다
    public void printInfo() {
        System.out.println("[" + getGrade() + "] " + name + " / " + email
                           + " / " + phone + " (혜택: " + getBenefit() + ")");
    }

    // 수정용 (이름·이메일·연락처 한 번에 교체)
    public void update(String name, String email, String phone) {
        this.name = name;
        this.email = email;
        this.phone = phone;
    }
}
```

**확인**: 컴파일 에러가 없으면 OK. (`new Member(...)`를 시도해 "추상 클래스는 못 만든다" 에러를 확인해 보세요.)

---

### Step 2. 일반 회원 (`NormalMember.java`)

**목표**: `Member`를 상속해 일반 등급 회원을 만든다.

**할 일**: `super(...)`로 부모 생성자 호출, `getGrade()`/`getBenefit()` 구현.

**힌트**

```java
public class NormalMember extends Member {
    public NormalMember(String name, String email, String phone) {
        super(name, email, phone);
    }
    @Override public String getGrade()   { return "일반"; }
    @Override public String getBenefit() { return "기본 서비스"; }
}
```

**확인**: 일반 회원을 만들어 `printInfo()` 하면 `[일반] ...` 형식으로 나오면 성공.

---

### Step 3. VIP 회원 (`VipMember.java`)

**목표**: 같은 패턴으로 VIP 등급 회원을 만든다 (혜택만 다르게).

**힌트**

```java
public class VipMember extends Member {
    public VipMember(String name, String email, String phone) {
        super(name, email, phone);
    }
    @Override public String getGrade()   { return "VIP"; }
    @Override public String getBenefit() { return "10% 할인 + 무료배송"; }
}
```

**확인**: VIP 회원의 `printInfo()`가 `[VIP] ... (혜택: 10% 할인 ...)`로 일반과 다르게 나오면 성공. (다형성!)

---

### Step 4. 회원 관리자 — 추가 (`MemberManager.java`)

**목표**: 회원 배열과 인원수를 관리하고, 회원을 추가한다.

**할 일**

1. 필드 `Member[] members`, `int memberCnt`. 생성자에서 정원만큼 배열 생성.
2. `isFull()` — 정원이 찼는지.
3. `existsEmail(String email)` — 이메일 중복 검사.
4. `add(Member m)` — 배열에 넣고 인원 +1.

**힌트**

```java
public class MemberManager {
    private Member[] members;
    private int memberCnt;

    public MemberManager(int capacity) {
        members = new Member[capacity];
        memberCnt = 0;
    }

    public boolean isFull() {
        return memberCnt == members.length;
    }

    public boolean existsEmail(String email) {
        for (int i = 0; i < memberCnt; i++) {        // memberCnt까지만 → null 걱정 없음
            if (members[i].getEmail().equals(email)) return true;
        }
        return false;
    }

    public void add(Member m) {
        members[memberCnt] = m;
        memberCnt++;
    }

    public int getCount()    { return memberCnt; }
    public int getCapacity() { return members.length; }
}
```

> **개선 포인트**: `memberCnt`까지만 순회하므로 빈 칸(`null`)을 만질 일이 없어, 원래의 null 검사가 필요 없어집니다.

**확인**: 매니저를 만들고 회원을 add 한 뒤 `getCount()`가 늘어나면 성공. (실제 추가 입력은 Step 7에서 연결)

---

### Step 5. 조회 (`MemberManager.java`)

**목표**: 이메일/이름으로 회원을 찾고, 전체를 출력한다.

**할 일**

1. `findByEmail(String)` / `findByName(String)` — 찾으면 그 `Member`를, 없으면 `null` 반환.
2. `printAll()` — 등록된 회원 전체를 `printInfo()`로 출력.

**힌트**

```java
public Member findByEmail(String email) {
    for (int i = 0; i < memberCnt; i++) {
        if (members[i].getEmail().equals(email)) return members[i];
    }
    return null;
}

public Member findByName(String name) {
    for (int i = 0; i < memberCnt; i++) {
        if (members[i].getName().equals(name)) return members[i];
    }
    return null;
}

public void printAll() {
    for (int i = 0; i < memberCnt; i++) {
        members[i].printInfo();
    }
}
```

> 조회 결과 출력은 `Main`에서 `Member m = manager.findByEmail(...)` 후 `m == null`이면 "없습니다", 아니면 `m.printInfo()` 하면 됩니다. (찾기와 출력의 역할 분리)

**확인**: 추가한 회원을 이메일/이름으로 찾으면 정보가 나오고, 전체조회가 동작하면 성공.

---

### Step 6. 수정 & 삭제 (`MemberManager.java`)

**목표**: 이메일로 찾아 정보를 수정하거나, 삭제하고 뒤를 당긴다.

**할 일**

1. `update(...)` — 이메일로 회원을 찾아 `member.update(새 값)` 호출.
2. `delete(String email)` — 찾아서 배열을 한 칸씩 당기고 마지막을 `null`, 인원 -1.

**힌트**

```java
public boolean update(String email, String name, String newEmail, String phone) {
    Member m = findByEmail(email);
    if (m == null) return false;
    m.update(name, newEmail, phone);   // 객체에게 수정을 맡김
    return true;
}

public boolean delete(String email) {
    int idx = -1;
    for (int i = 0; i < memberCnt; i++) {
        if (members[i].getEmail().equals(email)) { idx = i; break; }
    }
    if (idx == -1) return false;

    for (int i = idx; i < memberCnt - 1; i++) {
        members[i] = members[i + 1];   // 객체 하나만 당기면 됨 (이름·메일·연락처 따로 X)
    }
    members[memberCnt - 1] = null;
    memberCnt--;
    return true;
}
```

> **개선 포인트**: 원래는 삭제 시 `[0]/[1]/[2]` 세 칸을 각각 당겼지만, 이제 **객체 참조 한 번만** 당기면 됩니다.

**확인**: 수정 후 조회하면 바뀌고, 가운데 회원을 삭제해도 목록이 빈칸 없이 이어지면 성공.

---

### Step 7. 요금제 + 메뉴 루프 (`Main.java`)

**목표**: 요금제로 정원을 정하고, 메뉴를 반복하며 매니저 기능을 호출한다. 추가 시 등급을 골라 알맞은 회원 객체를 만든다.

**할 일**

1. 요금제(1~3)를 받아 정원 = 선택 × 10, `new MemberManager(정원)`.
2. `while` 메뉴 루프 + `switch`.
3. 회원추가: 등급([1]일반 [2]VIP)·이름·이메일·연락처를 받아 `NormalMember`/`VipMember` 생성 후 `add`.

**힌트**

```java
Scanner sc = new Scanner(System.in);
// 요금제
System.out.println("[1]Lite:10 [2]Basic:20 [3]Premium:30");
int plan = Integer.parseInt(sc.nextLine());
MemberManager manager = new MemberManager(plan * 10);

// 회원추가 부분 (case 1)
if (manager.isFull()) {
    System.out.println("회원이 꽉 찼습니다.");
} else {
    System.out.println("등급 [1]일반 [2]VIP");
    int grade = Integer.parseInt(sc.nextLine());
    System.out.print("이름 > ");   String name  = sc.nextLine();
    System.out.print("이메일 > "); String email = sc.nextLine();
    System.out.print("연락처 > "); String phone = sc.nextLine();

    if (manager.existsEmail(email)) {
        System.out.println("이미 존재하는 회원입니다.");
    } else {
        Member m = (grade == 2) ? new VipMember(name, email, phone)
                                : new NormalMember(name, email, phone);
        manager.add(m);   // 어떤 등급이든 Member로 추가 (다형성)
    }
}
```

> **팁**: 입력은 `Integer.parseInt(sc.nextLine())`로 받으면, 숫자와 줄바꿈이 꼬여 이름이 비는 흔한 버그를 피할 수 있어요. (`Scanner`는 하나만 만들어 공유)

**확인**: 등급을 골라 회원을 추가하고, 전체조회에서 등급·혜택이 다르게 보이면 성공!

### Step 8. 마무리 점검 (완성!)

**점검 항목**

- [ ] `Member`는 `abstract`이고 직접 `new` 할 수 없는가?
- [ ] 일반/VIP가 `getBenefit()`을 다르게 구현했는가?
- [ ] `Member[]` 한 배열에 일반·VIP가 섞여 담기는가? (다형성)
- [ ] 회원 데이터·CRUD가 `MemberManager` 안에 있는가? (캡슐화)
- [ ] 정원 초과·이메일 중복·없는 회원 처리가 되는가?
- [ ] 삭제 후 목록이 빈칸 없이 이어지는가?

여기까지 통과하면 **OOP 회원 관리 완성입니다!** 🎉

## 8. (선택) 도전 과제

1. **인터페이스 적용**: 정렬을 위해 `Comparable<Member>`를 구현해 이름순/등급순 정렬
2. **등급 추가**: VVIP 등 새 등급을 클래스 하나 추가로 확장 (다형성의 장점 체감)
3. **등급별 동작 차이 키우기**: `getBenefit()` 외에 `getMonthlyFee()` 같은 추상 메서드를 더해 등급별 요금 계산
4. **컬렉션으로**: 고정 배열 대신 `ArrayList<Member>`로 바꿔 정원·당기기 코드 제거
5. **파일 저장**: 프로그램을 꺼도 회원이 유지되도록 파일 입출력 추가
