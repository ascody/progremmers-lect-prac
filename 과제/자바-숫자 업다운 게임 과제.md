# 숫자 업다운 게임 만들기 (Up & Down)

> 컴퓨터가 정한 숫자를 사용자가 맞히는 콘솔 게임입니다. 자바의 기본기 — `Random`, `Scanner`, `if-else`, `while` 반복 — 만으로 만들 수 있어요. **아래 Step을 순서대로 따라가면, 마지막 Step에서 게임이 완성됩니다.**

---

## 1. 무엇을 만드나요?

컴퓨터가 1~100 중 하나를 몰래 정합니다. 사용자가 숫자를 입력하면 "더 큰 수(UP)" 또는 "더 작은 수(DOWN)"라고 알려주고, 맞힐 때까지 반복합니다.

```
숫자를 맞혀보세요! (1 ~ 100)
입력 > 50
UP! 더 큰 수입니다.
입력 > 75
DOWN! 더 작은 수입니다.
입력 > 63
DOWN! 더 작은 수입니다.
입력 > 58
정답입니다! 4번 만에 맞혔어요.
```

---

## 2. 요구사항 정리 (기능 명세)

|번호|기능|설명|
|---|---|---|
|1|정답 정하기|프로그램이 1~100 사이의 **랜덤 숫자**를 하나 정한다|
|2|숫자 입력|사용자에게 숫자를 입력받는다|
|3|힌트 주기|입력값이 정답보다 작으면 **UP**, 크면 **DOWN**을 알려준다|
|4|반복|정답을 맞힐 때까지 2~3을 반복한다|
|5|시도 횟수|몇 번 만에 맞혔는지 세어서 보여준다|

---

## 3. 핵심 개념

### (1) 랜덤 숫자 만들기 — `Random`

```java
Random rand = new Random();
int answer = rand.nextInt(100) + 1;   // 0~99 에 +1 → 1~100
```

> `nextInt(100)`은 0~99를 주므로, 1~100을 원하면 **+1**을 해줍니다.

### (2) 비교해서 힌트 주기 — `if-else`

입력값(`guess`)과 정답(`answer`)을 비교합니다.

- `guess < answer` → 정답이 더 큼 → **UP**
- `guess > answer` → 정답이 더 작음 → **DOWN**
- `guess == answer` → **정답!**

### (3) 맞힐 때까지 반복 — `while`

정답을 맞히기 전까지 입력·비교를 계속 돌리고, 맞으면 `break`로 빠져나옵니다.

---

## 4. Step by Step

각 Step에 **목표 / 할 일 / 힌트 / 확인 방법**이 있습니다. 하나씩 실행해 보며 진행하세요. (전체를 `main` 하나에 작성해도 충분합니다.)

---

### Step 1. 정답 정하고, 숫자 한 번 입력받기

**목표**: 컴퓨터가 랜덤 정답을 정하고, 사용자에게 숫자를 한 번 입력받는다.

**할 일**

1. `Random`으로 1~100 정답을 만든다.
2. `Scanner`로 숫자 하나를 입력받는다.
3. (테스트용) 정답을 잠깐 화면에 출력해 둔다. → 완성 후 지운다.

**힌트**

```java
import java.util.Random;
import java.util.Scanner;

Random rand = new Random();
int answer = rand.nextInt(100) + 1;
System.out.println("(테스트용 정답: " + answer + ")"); // 나중에 삭제!

Scanner sc = new Scanner(System.in);
System.out.print("입력 > ");
int guess = sc.nextInt();
```

**확인**: 실행할 때마다 정답이 바뀌고, 내가 입력한 숫자가 잘 들어오면 성공.

---

### Step 2. 입력값과 정답 비교하기 (한 번)

**목표**: 입력한 숫자가 정답보다 큰지 작은지 같은지 판정한다.

**할 일**: `if-else`로 세 경우(UP / DOWN / 정답)를 나눈다.

**힌트**

```java
if (guess < answer) {
    System.out.println("UP! 더 큰 수입니다.");
} else if (guess > answer) {
    System.out.println("DOWN! 더 작은 수입니다.");
} else {
    System.out.println("정답입니다!");
}
```

**확인**: 정답보다 작게/크게/똑같이 넣어보고 메시지가 맞게 나오면 성공.

---

### Step 3. 맞힐 때까지 반복하기

**목표**: 한 번만 비교하지 말고, 정답을 맞힐 때까지 계속 입력받는다.

**할 일**

1. 입력과 비교를 `while (true)` 안으로 넣는다.
2. 정답일 때 `break`로 반복을 끝낸다.

**힌트**

```java
while (true) {
    System.out.print("입력 > ");
    int guess = sc.nextInt();

    if (guess == answer) {
        System.out.println("정답입니다!");
        break;                       // 맞히면 반복 종료
    } else if (guess < answer) {
        System.out.println("UP!");
    } else {
        System.out.println("DOWN!");
    }
}
```

**확인**: 맞힐 때까지 계속 물어보고, 맞히면 멈추면 성공.

---

### Step 4. 시도 횟수 세기

**목표**: 몇 번 만에 맞혔는지 알려준다.

**할 일**

1. 반복 밖에 `int count = 0;`을 둔다.
2. 입력받을 때마다 `count++`.
3. 정답일 때 `count`를 함께 출력한다.

**힌트**

```java
int count = 0;
while (true) {
    int guess = sc.nextInt();
    count++;                         // 시도 횟수 +1
    if (guess == answer) {
        System.out.println("정답입니다! " + count + "번 만에 맞혔어요.");
        break;
    }
    // ... UP / DOWN
}
```

**확인**: 맞힌 뒤 "○번 만에 맞혔어요"가 정확히 나오면 성공.

---

### Step 5. 마무리 다듬기 (완성!)

**목표**: 사용자가 쓰기 편하게 작은 부분들을 보완한다.

**점검 항목**

- [ ] 시작할 때 범위(1~100)를 안내하는가?
- [ ] 테스트용 정답 출력을 **지웠는가?**
- [ ] 1~100 밖의 숫자를 넣으면 "범위를 벗어났어요" 안내가 나오는가? (선택)
- [ ] (선택) 숫자가 아닌 글자를 입력해도 프로그램이 죽지 않는가?

**힌트 (범위 안내 예시)**

```java
if (guess < 1 || guess > 100) {
    System.out.println("1~100 사이로 입력해 주세요.");
    continue;            // 이번 입력은 무효 처리, 다시 입력받기
}
```

여기까지 통과하면 **업다운 게임 완성입니다!** 🎉
