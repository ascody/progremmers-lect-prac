# 가계부 만들기 (Collections 활용)

> Java의 인터페이스, 구현 클래스, 그리고 `Map` / `List` 같은 컬렉션을 사용해 간단한 가계부 프로그램을 완성합니다. **아래 Step을 순서대로 따라가면, 마지막 Step에서 프로그램이 완성됩니다.**

---

## 1. 무엇을 만드나요?

콘솔(터미널)에서 동작하는 가계부 프로그램입니다. 사용자가 날짜별로 지출 내역(예: 공책 1000원, 연필 300원)을 기록하고, 조회하고, 삭제할 수 있습니다.

실행하면 아래와 같은 메뉴가 반복해서 나오고, 번호를 입력해 기능을 사용합니다.

```
===== 가계부 =====
1. 내역 추가
2. 내역 조회
3. 전체 삭제
4. 내역 삭제
5. 종료
번호 입력 >
```

---

## 2. 요구사항 정리 (기능 명세)

|번호|기능|설명|
|---|---|---|
|1|내역 추가|날짜를 입력하고, 그 날짜에 항목(이름 + 금액)을 여러 개 등록한다. 등록이 끝나면 합계를 보여준다.|
|2|내역 조회|지금까지 기록한 **날짜 목록**을 보여준다. 사용자가 날짜 하나를 입력하면 그 날의 항목들과 합계를 보여준다.|
|3|전체 삭제|날짜를 입력받아 **그 날짜와 모든 항목**을 통째로 삭제한다.|
|4|내역 삭제|날짜를 입력받아 항목들을 보여주고, 그중 **특정 항목 하나**만 삭제한다.|
|5|종료|프로그램을 끝낸다.|

### 기능 1, 2 동작 예시

**내역 추가**

```
날짜 입력 (예: 2024-09-04) > 2024-09-04
항목 이름 > 공책
금액 > 1000
더 추가할까요? (y/n) > y
항목 이름 > 연필
금액 > 300
더 추가할까요? (y/n) > n

[2024-09-04] 등록 완료
공책 : 1000원
연필 : 300원
합계 : 1300원
```

**내역 조회**

```
== 기록된 날짜 ==
2024-09-04
2024-09-03
2024-09-02
2024-09-01
조회할 날짜 입력 > 2024-09-04

[2024-09-04]
공책 : 1000원
연필 : 300원
합계 : 1300원
```

## 3. 핵심 개념: 데이터를 어떻게 저장할까?

가계부는 **"날짜 → 그 날짜의 항목 목록"** 구조입니다. 이걸 컬렉션으로 표현하면 다음과 같습니다.

```
Map<String, List<Item>>
      │           │
   날짜 문자열   그 날의 항목들(List)
```

- 날짜(`"2024-09-04"`)를 **키(key)**, 항목 목록을 **값(value)**으로 갖는 `Map`
- 한 날짜에 항목이 여러 개일 수 있으므로 값은 `List`
- 항목 하나는 **이름 + 금액**을 함께 가지므로 작은 클래스 `Item`으로 묶습니다

> 이 과제의 핵심은 바로 이 `Map`과 `List`를 다루는 것입니다. (추가/조회 = `put`/`get`, 삭제 = `remove`, 목록 = `keySet`)

## 4. 파일 구조 (각 파일의 역할)

|파일|역할|
|---|---|
|`Item.java`|항목 1개를 표현하는 클래스 (이름, 금액) — _도우미 클래스_|
|`AccountBook.java` _(인터페이스)_|가계부가 "할 수 있는 기능"을 **선언**만 한다 (메서드 이름 목록)|
|`AccountBookImpl.java`|`AccountBook`을 **구현**한다. 실제 로직과 데이터(`Map`)를 가진다|
|`Start.java`|`main` 메서드. 메뉴를 출력하고 사용자 입력에 따라 기능을 호출한다|

> 인터페이스(`AccountBook`)는 "무엇을 하는지"만, 구현 클래스(`AccountBookImpl`)는 "어떻게 하는지"를 담당합니다. 이 분리를 직접 경험하는 것이 과제의 목표 중 하나입니다.

## 5. Step by Step

각 Step에는 **목표 / 할 일 / 힌트 / 확인 방법**이 있습니다. 한 Step을 끝내고 실행이 잘 되는지 확인한 뒤 다음 Step으로 넘어가세요.

### Step 1. 메뉴부터 띄우기 (`Start.java`)

**목표**: 아직 기능이 없어도, 메뉴가 반복 출력되고 `5`를 누르면 종료되게 만든다.

**할 일**

- `Scanner`로 번호를 입력받는다.
- `while` 반복문 안에서 메뉴를 출력한다.
- `switch` 또는 `if`로 번호를 구분한다.
- `5`를 입력하면 반복을 멈추고 "프로그램을 종료합니다" 출력.

**힌트**

```java
Scanner sc = new Scanner(System.in);
while (true) {
    System.out.println("===== 가계부 =====");
    // 1~5 메뉴 출력
    int menu = Integer.parseInt(sc.nextLine());
    switch (menu) {
        case 1: /* 나중에 추가 */ break;
        // ...
        case 5: System.out.println("종료합니다"); return;
        default: System.out.println("잘못된 번호입니다");
    }
}
```

**확인**: 1~4를 누르면 아무 일도 안 일어나고, 5를 누르면 종료되면 성공.

### Step 2. 항목 클래스 만들기 (`Item.java`)

**목표**: 항목 하나(이름 + 금액)를 담는 클래스를 만든다.

**할 일**

- 필드 `String name`, `int price`
- 생성자
- 값을 꺼낼 수 있는 getter (또는 출력용 메서드)

**힌트**

```java
public class Item {
    private String name;
    private int price;

    public Item(String name, int price) {
        this.name = name;
        this.price = price;
    }
    public String getName()  { return name; }
    public int getPrice()    { return price; }
}
```

**확인**: 컴파일 에러가 없으면 OK. (아직 화면에 보이는 건 없습니다)

---

### Step 3. 인터페이스 작성 (`AccountBook.java`)

**목표**: 가계부의 4가지 기능을 메서드로 **선언**한다. (구현 X)

**할 일**

- 기능 5개 중 "종료"는 메뉴 로직이므로, 기능 1~4를 메서드로 선언한다.

**힌트** (추천 설계 — 입력/출력은 구현 클래스가 직접 처리하는 방식)

```java
public interface AccountBook {
    void addAccount();    // 1. 내역 추가
    void showAccount();   // 2. 내역 조회
    void deleteAll();     // 3. 전체 삭제
    void deleteItem();    // 4. 내역 삭제
}
```

**확인**: 인터페이스에는 `{ }` 본문 없이 `;`로 끝나는 메서드 선언만 있으면 됩니다.

### Step 5. 기능 1 — 내역 추가 (`addAccount`)

**목표**: 날짜를 받고, 항목을 여러 개 입력받아 `Map`에 저장한다. 끝나면 합계를 출력한다.

**할 일**

1. 날짜를 입력받는다.
2. 그 날짜에 쓸 빈 `List<Item>`을 만든다.
3. 반복하면서 항목 이름·금액을 입력받아 `Item`을 만들고 `List`에 `add`한다.
4. "더 추가?"가 `n`이면 반복 종료.
5. `data.put(날짜, 리스트)`로 저장한다.
6. 항목들과 합계를 출력한다.

**힌트**

- 합계는 리스트를 돌면서 `getPrice()`를 더하면 됩니다.
- 같은 날짜를 또 입력하면? → 이미 있으면 기존 리스트에 더할지, 덮어쓸지 정책을 정하세요. (간단히는 `containsKey`로 확인 후 기존 리스트에 `add`)

**확인**: 추가 후 화면에 항목과 합계가 예시처럼 나오면 성공.

---

### Step 6. 기능 2 — 내역 조회 (`showAccount`)

**목표**: 날짜 목록을 보여주고, 선택한 날짜의 항목과 합계를 출력한다.

**할 일**

1. `data`가 비어 있으면 "기록이 없습니다" 출력 후 종료.
2. `data.keySet()`으로 날짜들을 모두 출력한다.
3. 날짜 하나를 입력받는다.
4. `data.containsKey(날짜)`로 있는지 확인.
    - 없으면 "그런 날짜가 없습니다" 출력.
    - 있으면 `data.get(날짜)`의 항목들과 합계를 출력.

**힌트 (날짜를 최신순으로 정렬해 보여주려면)**

```java
List<String> dates = new ArrayList<>(data.keySet());
Collections.sort(dates, Collections.reverseOrder()); // 2024-09-04 → ... 순
for (String d : dates) System.out.println(d);
```

**확인**: 날짜 목록이 나오고, 날짜를 입력하면 그 날의 내역이 나오면 성공.

---

### Step 7. 기능 3 — 전체 삭제 (`deleteAll`)

**목표**: 입력한 날짜와 그 날의 모든 항목을 통째로 지운다.

**할 일**

1. 날짜를 입력받는다.
2. `data.containsKey(날짜)`로 확인.
3. 있으면 `data.remove(날짜)` 후 "삭제되었습니다" 출력.
4. 없으면 안내 메시지.

**힌트**: `Map.remove(key)` 한 줄이면 그 키와 값(리스트 전체)이 사라집니다.

**확인**: 삭제 후 "내역 조회"하면 그 날짜가 목록에서 사라져 있으면 성공.

---

### Step 8. 기능 4 — 내역 삭제 (`deleteItem`)

**목표**: 한 날짜 안에서 항목 하나만 골라 삭제한다.

**할 일**

1. 날짜를 입력받고 존재하는지 확인.
2. 그 날의 항목들을 **번호와 함께** 출력한다.
    
    ```
    1. 공책 : 1000원2. 연필 : 300원
    ```
    
3. 삭제할 번호를 입력받는다.
4. `List.remove(번호 - 1)`로 해당 항목을 지운다.
5. (선택) 그 날의 항목이 0개가 되면 날짜 자체도 지운다.

**힌트**

- 화면 번호는 1부터, 리스트 인덱스는 0부터이므로 `remove(번호 - 1)`.
- 잘못된 번호를 막으려면 입력값이 `1 ~ list.size()` 범위인지 검사하세요.

**확인**: 특정 항목만 사라지고 나머지는 그대로 남아 있으면 성공.

## 7. (선택) 도전 과제

여유가 있는 학생은 아래 중 골라서 확장해 보세요.

1. **정렬**: 날짜 목록을 항상 최신순으로 자동 정렬 (`TreeMap` 활용)
2. **총 지출 합계**: 모든 날짜의 금액을 합쳐 "이번 달 총 지출" 출력
3. **수정 기능**: 특정 항목의 금액을 바꾸는 기능 추가
4. **수입/지출 구분**: `Item`에 종류(수입/지출) 필드를 추가하고 잔액 계산
5. **파일 저장**: 프로그램을 꺼도 데이터가 남도록 파일에 저장/불러오기
