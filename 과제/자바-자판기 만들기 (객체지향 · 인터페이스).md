# 자판기 만들기 (객체지향 · 인터페이스)

> 자판기를 **인터페이스(`interface`) + 구현(`implements`) + 다형성**으로 설계합니다. 추상 클래스 버전과 기능은 똑같지만, 음료의 "규약"을 인터페이스 `Drink`로 정하고 각 음료가 그 규약을 직접 구현합니다. **아래 Step을 순서대로 따라가면, 마지막 Step에서 프로그램이 완성됩니다.** 각 음료가 이 규약을 구현해 동작합니다.

---

## 1. 무엇을 만드나요?

돈을 넣고 번호로 음료를 고르면 음료가 나오고 금액이 차감됩니다. 종료 시 잔돈을 돌려줍니다.

```
============== 자판기 ==============
[1]콜라 : 500  [2]사이다 : 500  [3]환타 : 300  [4]물 : 200
[5]돈 넣기  [6]종료
현재 금액 : 0
====================================
원하는 메뉴를 선택하세요 > 5
넣을 금액 > 1000
1000원을 넣었습니다.
...
원하는 메뉴를 선택하세요 > 2
톡 쏘는 사이다가 나왔습니다!
```

---

## 2. 학습 목표

|개념|어디서 배우나|
|---|---|
|인터페이스|음료의 규약 `Drink` (메서드 목록만)|
|구현 (`implements`)|각 음료가 규약을 구현|
|오버라이딩 (`@Override`)|음료마다 `dispense()`를 다르게 구현|
|다형성|`Drink[]`에 여러 음료를 담고 똑같이 다룬다|
|**추상 클래스 vs 인터페이스**|같은 자판기를 두 방식으로 만들어 차이를 비교|

---

## 3. 핵심 개념

### (1) 인터페이스 = "할 수 있는 일"의 목록(규약)

인터페이스는 **필드(상태)도 생성자도 가질 수 없습니다.** 오직 "이런 메서드를 갖춰라"는 약속만 합니다.

```java
public interface Drink {
    String getName();
    int getPrice();
    void dispense();
}
```

이 인터페이스를 `implements` 하는 클래스는 **세 메서드를 모두 반드시 구현**해야 합니다.

### (2) 추상 클래스 버전과의 차이

||추상 클래스 `Drink`|인터페이스 `Drink`|
|---|---|---|
|필드(name/price)|부모에 **한 번만** 둠|각 음료가 **직접** 가짐|
|생성자|있음 (`super(...)` 호출)|**없음**|
|getName/getPrice|부모에 한 번 구현 → 물려받음|각 음료가 **각자** 구현|
|다중 적용|단일 상속만|여러 개 동시 구현 가능|

> 즉 인터페이스는 공통 코드를 물려줄 수 없어서, 각 음료 클래스가 더 **자기완결적**이 됩니다. 대신 "여러 규약을 동시에 갖춘다"가 가능합니다. (예: `implements Drink, Comparable`)

### (3) 다형성 — 사용하는 쪽은 차이가 안 보인다

```java
Drink[] drinks = { new Coke(), new Cider(), new Fanta(), new Water() };
drinks[1].dispense();   // 실제로는 Cider의 dispense()가 실행됨
```

자판기 입장에선 추상 클래스든 인터페이스든 똑같이 `Drink`로 다룹니다.

## 4. 파일 구조

|파일|역할|
|---|---|
|`Drink.java`|**인터페이스**. 음료가 갖춰야 할 메서드 목록|
|`Coke.java` / `Cider.java` / `Fanta.java` / `Water.java`|`Drink`를 구현한 실제 음료들|
|`VendingMachine.java`|돈·음료를 관리 (넣기/메뉴/구매/잔돈)|
|`Main.java`|메뉴를 받아 자판기 기능을 호출|

## 5. Step by Step

각 Step에 **목표 / 할 일 / 힌트 / 확인 방법**이 있습니다.

---

### Step 1. 음료 규약 — 인터페이스 (`Drink.java`)

**목표**: 모든 음료가 갖춰야 할 메서드 3개를 인터페이스로 선언한다.

**할 일**

1. `interface Drink` 선언.
2. `getName()`, `getPrice()`, `dispense()`를 **본문 없이** 선언.

**힌트**

```java
public interface Drink {
    String getName();
    int getPrice();
    void dispense();
}
```

> 인터페이스의 메서드는 자동으로 `public abstract`입니다. 필드나 생성자는 넣지 않습니다.

**확인**: 컴파일 에러가 없으면 OK.

---

### Step 2. 첫 음료 — 콜라 (`Coke.java`)

**목표**: `Drink`를 구현한 콜라 클래스를 만든다. (이름·가격을 **직접** 가진다)

**할 일**

1. `class Coke implements Drink`.
2. 필드 `name`, `price`를 직접 선언하고 값을 준다.
3. `getName()`, `getPrice()`, `dispense()`를 **모두** 구현.

**힌트**

```java
public class Coke implements Drink {
    private String name = "콜라";
    private int price = 500;

    @Override public String getName() { return name; }
    @Override public int getPrice()   { return price; }
    @Override public void dispense() {
        System.out.println("시원한 콜라가 나왔습니다!");
    }
}
// Main 에서 임시 테스트
Drink c = new Coke();          // 인터페이스 타입에 구현 객체 담기 (다형성)
System.out.println(c.getName() + " " + c.getPrice());  // 콜라 500
c.dispense();                      // 시원한 콜라가 나왔습니다!
```

> 추상 클래스 버전과 비교해 보세요. 거기선 `super("콜라", 500)` 한 줄이면 됐지만, 여기선 이름·가격·getter를 **이 클래스가 직접** 다 갖춰야 합니다. 그게 인터페이스의 특징이에요.

**확인**: "콜라 500"과 콜라 메시지가 나오면 성공.

---

### Step 3. 나머지 음료들 (`Cider`, `Fanta`, `Water`)

**목표**: 콜라와 같은 패턴으로 사이다·환타·물을 만든다.

**할 일**: `Coke`를 본떠 이름·가격·메시지만 바꿔 세 클래스를 만든다.

**힌트**

|클래스|name / price|dispense() 메시지|
|---|---|---|
|`Cider`|"사이다" / 500|톡 쏘는 사이다가 나왔습니다!|
|`Fanta`|"환타" / 300|달콤한 환타가 나왔습니다!|
|`Water`|"물" / 200|깔끔한 물이 나왔습니다!|

**확인**: 네 음료 모두 `implements Drink`로 컴파일되고, 각각 `dispense()`가 다르게 나오면 성공.

---

### Step 4. 자판기 기본기 — 돈 넣기 & 메뉴 (`VendingMachine.java`)

**목표**: 돈과 음료 목록을 관리하는 자판기 클래스를 만든다.

**할 일**

1. 필드 `totalMoney`, `Drink[] drinks`.
2. 생성자에서 `drinks`에 음료 4개를 담는다 (다형성).
3. `insertMoney(int)`, `printMenu()` 작성.

**힌트**

```java
public class VendingMachine {
    private int totalMoney;
    private Drink[] drinks;

    public VendingMachine() {
        totalMoney = 0;
        drinks = new Drink[] {        // 인터페이스 타입 배열에 구현 객체들을 담음
            new Coke(), new Cider(), new Fanta(), new Water()
        };
    }

    public void insertMoney(int money) {
        totalMoney += money;
        System.out.println(money + "원을 넣었습니다.");
    }

    public void printMenu() {
        System.out.println("============== 자판기 ==============");
        System.out.println("[1]콜라 : 500  [2]사이다 : 500  [3]환타 : 300  [4]물 : 200");
        System.out.println("[5]돈 넣기  [6]종료");
        System.out.println("현재 금액 : " + totalMoney);
        System.out.println("====================================");
    }
}
```

**확인**: 자판기를 만들어 `printMenu()`를 부르면 메뉴가 출력되면 성공.

---

### Step 5. 핵심 — 구매와 잔돈 반환 (`VendingMachine.java`)

**목표**: 번호로 음료를 사고(금액 차감), 종료 시 잔돈을 돌려준다.

**할 일**

1. `buy(int menuNumber)`: 해당 음료를 꺼내 잔돈 검사 → 차감 → `dispense()`.
2. `returnChange()`: 현재 금액을 돌려주고 0으로 초기화.

**힌트**

```java
public void buy(int menuNumber) {
    Drink drink = drinks[menuNumber - 1];   // 어떤 음료든 Drink로 받음

    if (totalMoney < drink.getPrice()) {
        System.out.println("잔돈이 부족합니다.");
        return;
    }
    totalMoney -= drink.getPrice();
    drink.dispense();   // 선택된 음료의 dispense()가 실행됨 (다형성)
}

public int returnChange() {
    int change = totalMoney;
    totalMoney = 0;
    return change;
}
```

> 이 부분은 추상 클래스 버전과 **완전히 동일**합니다. 사용하는 쪽에선 인터페이스인지 추상 클래스인지 차이가 없으니까요.

**확인**: (다음 Step에서 메뉴와 연결해 확인)

---

### Step 6. 메뉴 루프로 연결 (`Main.java`)

**목표**: 메뉴를 반복 출력하고, 입력에 따라 자판기 기능을 호출한다.

**힌트**

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        VendingMachine machine = new VendingMachine();

        while (true) {
            machine.printMenu();
            System.out.print("원하는 메뉴를 선택하세요 > ");
            int choice = sc.nextInt();

            if (choice >= 1 && choice <= 4) {
                machine.buy(choice);
            } else if (choice == 5) {
                System.out.print("넣을 금액 > ");
                machine.insertMoney(sc.nextInt());
            } else if (choice == 6) {
                System.out.println("\n잔돈 " + machine.returnChange() + "원이 반환되었습니다.");
                return;
            } else {
                System.out.println("잘못 입력하셨습니다. 다시 입력해주세요.");
            }
        }
    }
}
```

**확인**: 돈 넣고 음료 사고, 잔돈 부족도 처리되고, 종료 시 잔돈이 반환되면 성공!

## 8. (선택) 도전 과제

1. **두 버전 비교 정리**: 추상 클래스 버전과 인터페이스 버전을 표로 비교해 보기 (필드/생성자 유무, 코드 중복, 다중 구현 가능 여부)
2. **다중 구현**: `Drink`에 더해 `Comparable<Drink>` 같은 인터페이스를 함께 구현해 가격순 정렬해 보기 (인터페이스만의 장점!)
3. **default 메서드**: Java 8 `default` 메서드로 공통 동작을 인터페이스에 넣어 보기
4. **재고/품절**: 음료별 남은 수량을 두고 0이면 "품절" 처리
5. **음료 추가**: 커피 클래스를 추가하고 배열에 한 줄만 더해 동작하는지 확인

### 언제 무엇을 쓸까? (한 줄 정리)

- **상태(필드)나 공통 구현을 물려주고 싶다** → 추상 클래스
- **"이런 기능을 갖춰라"는 규약만 강제하고, 여러 개를 동시에 적용하고 싶다** → 인터페이스
