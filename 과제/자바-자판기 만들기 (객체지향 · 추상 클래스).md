# 자판기 만들기 (객체지향 · 추상 클래스)

> 절차지향으로 짠 자판기를 **추상 클래스(`abstract`) + 상속 + 다형성**으로 다시 설계합니다. 음료의 공통 부분은 추상 클래스 `Drink`에 모으고, 음료마다 다른 동작은 추상 메서드로 강제해 각 음료가 구현합니다. **아래 Step을 순서대로 따라가면, 마지막 Step에서 프로그램이 완성됩니다.**

---

## 1. 무엇을 만드나요?

돈을 넣고, 번호로 음료를 고르면 음료가 나오고 금액이 차감됩니다. 종료하면 잔돈을 돌려줍니다.

```
============== 자판기 ==============
[1]콜라 : 500  [2]사이다 : 500  [3]환타 : 300  [4]물 : 200
[5]돈 넣기  [6]종료
현재 금액 : 0
====================================
원하는 메뉴를 선택하세요 > 5
넣을 금액 > 1000
1000원을 넣었습니다.
...
원하는 메뉴를 선택하세요 > 1
시원한 콜라가 나왔습니다!
```

> 기능은 절차지향 버전과 똑같습니다. **다른 건 "구조"** — 거대한 `switch` 대신 객체들이 역할을 나눠 갖습니다.

---

## 2. 학습 목표

|개념|어디서 배우나|
|---|---|
|추상 클래스 / 추상 메서드|음료의 공통 틀 `Drink`, 강제되는 `dispense()`|
|상속 (`extends`, `super`)|각 음료가 `Drink`를 물려받아 만들어진다|
|오버라이딩 (`@Override`)|음료마다 `dispense()`를 다르게 구현|
|다형성|`Drink[]`에 여러 음료를 담고 똑같이 다룬다|
|캡슐화|돈·로직을 `VendingMachine` 안으로 숨긴다|

---

## 3. 핵심 개념

### (1) 추상 클래스 = 공통 틀 + 강제 규약

모든 음료는 **이름·가격**을 공통으로 가집니다 → 부모 `Drink`에 한 번만 작성. 하지만 "나올 때의 동작"은 음료마다 다릅니다 → 내용 없이 `abstract`로 **선언만** 하고, 자식이 반드시 구현하게 강제합니다.

```java
public abstract void dispense();   // 본문 {}이 없음 → 자식이 채워야 함
```

> 추상 클래스는 `new Drink()`로 직접 만들 수 없습니다. 반드시 자식(`Coke` 등)을 통해 사용합니다.

### (2) 다형성 = 부모 타입으로 여러 자식을 똑같이 다루기

```java
Drink[] drinks = { new Coke(), new Cider(), new Fanta(), new Water() };
drinks[0].dispense();   // 실제로는 Coke의 dispense()가 실행됨
```

배열 타입은 `Drink` 하나지만, 실제로는 각기 다른 음료가 들어있고, `dispense()`를 부르면 **그 음료에 맞는 동작**이 실행됩니다. 절차지향의 `switch case 1~4` 중복이 이걸로 사라집니다.

## 5. Step by Step

각 Step에 **목표 / 할 일 / 힌트 / 확인 방법**이 있습니다.

---

### Step 1. 음료의 공통 틀 — 추상 클래스 (`Drink.java`)

**목표**: 모든 음료가 공유하는 이름·가격과, 강제할 동작 `dispense()`를 가진 추상 클래스를 만든다.

**할 일**

1. `abstract class Drink` 선언.
2. 필드 `name`, `price` (자식이 쓰도록 `protected`).
3. 이름·가격을 받는 생성자 (`this` 사용).
4. `getName()`, `getPrice()` 작성.
5. 본문 없는 추상 메서드 `dispense()` 선언.

**힌트**

```java
public abstract class Drink {
    protected String name;
    protected int price;

    public Drink(String name, int price) {
        this.name = name;
        this.price = price;
    }

    public String getName() { return name; }
    public int getPrice()   { return price; }

    public abstract void dispense();   // 자식이 반드시 구현
}
```

**확인**: 컴파일 에러가 없으면 OK. (`new Drink(...)`는 일부러 해보고 "추상 클래스는 못 만든다"는 에러를 확인해 보세요.)

---

### Step 2. 첫 번째 음료 — 콜라 (`Coke.java`)

**목표**: `Drink`를 상속한 콜라 클래스를 만들고, `dispense()`를 구현한다.

**할 일**

1. `class Coke extends Drink`.
2. 생성자에서 `super("콜라", 500)`로 부모 생성자 호출.
3. `dispense()`를 오버라이딩해 콜라만의 메시지 출력.

**힌트**

```java
public class Coke extends Drink {
    public Coke() {
        super("콜라", 500);   // 부모 생성자로 이름·가격 설정
    }
    @Override
    public void dispense() {
        System.out.println("시원한 콜라가 나왔습니다!");
    }
}
// Main 에서 임시 테스트
Drink c = new Coke();              // 부모 타입에 자식 담기 (다형성 맛보기)
System.out.println(c.getName() + " " + c.getPrice());  // 콜라 500
c.dispense();                      // 시원한 콜라가 나왔습니다!
```

**확인**: "콜라 500"과 콜라 메시지가 나오면 성공.

---

### Step 3. 나머지 음료들 (`Cider`, `Fanta`, `Water`)

**목표**: 콜라와 같은 패턴으로 사이다·환타·물을 만든다.

**할 일**: `Coke`를 본떠 가격과 메시지만 바꿔 세 클래스를 만든다.

**힌트**

|클래스|super(...)|dispense() 메시지|
|---|---|---|
|`Cider`|`super("사이다", 500)`|톡 쏘는 사이다가 나왔습니다!|
|`Fanta`|`super("환타", 300)`|달콤한 환타가 나왔습니다!|
|`Water`|`super("물", 200)`|깔끔한 물이 나왔습니다!|

**확인**: 네 음료 모두 컴파일되고, 각각 만들어 `dispense()`가 다르게 나오면 성공.

---

### Step 4. 자판기 기본기 — 돈 넣기 & 메뉴 (`VendingMachine.java`)

**목표**: 돈과 음료 목록을 관리하는 자판기 클래스를 만든다.

**할 일**

1. 필드 `totalMoney`, `Drink[] drinks`.
2. 생성자에서 `drinks`에 음료 4개를 담는다 (다형성!).
3. `insertMoney(int)` — 금액을 더한다.
4. `printMenu()` — 메뉴와 현재 금액을 출력.

**힌트**

```java
public class VendingMachine {
    private int totalMoney;
    private Drink[] drinks;

    public VendingMachine() {
        totalMoney = 0;
        drinks = new Drink[] {           // 부모 타입 배열에 자식들을 담음
            new Coke(), new Cider(), new Fanta(), new Water()
        };
    }

    public void insertMoney(int money) {
        totalMoney += money;
        System.out.println(money + "원을 넣었습니다.");
    }

    public void printMenu() {
        System.out.println("============== 자판기 ==============");
        System.out.println("[1]콜라 : 500  [2]사이다 : 500  [3]환타 : 300  [4]물 : 200");
        System.out.println("[5]돈 넣기  [6]종료");
        System.out.println("현재 금액 : " + totalMoney);
        System.out.println("====================================");
    }
}
```

**확인**: 자판기를 만들어 `printMenu()`를 부르면 메뉴가 출력되면 성공.

---

### Step 5. 핵심 — 구매와 잔돈 반환 (`VendingMachine.java`)

**목표**: 번호로 음료를 사고(금액 차감), 종료 시 잔돈을 돌려주는 기능을 만든다.

**할 일**

1. `buy(int menuNumber)`:
    - 번호로 해당 음료를 꺼낸다(`drinks[menuNumber - 1]`).
    - 돈이 부족하면 "잔돈이 부족합니다." 출력 후 종료.
    - 충분하면 가격만큼 차감하고 `dispense()` 호출.
2. `returnChange()`: 현재 금액을 돌려주고 0으로 초기화.

**힌트**

```java
public void buy(int menuNumber) {
    Drink drink = drinks[menuNumber - 1];   // 어떤 음료든 Drink로 받음

    if (totalMoney < drink.getPrice()) {
        System.out.println("잔돈이 부족합니다.");
        return;
    }
    totalMoney -= drink.getPrice();
    drink.dispense();   // 선택된 음료의 dispense()가 실행됨 (다형성!)
}

public int returnChange() {
    int change = totalMoney;
    totalMoney = 0;
    return change;
}
```

> **여기가 핵심**: 절차지향 버전의 `case 1~4`가 거의 똑같이 반복됐는데, 다형성 덕분에 `buy()` 하나로 모든 음료를 처리합니다.

**확인**: (다음 Step에서 메뉴와 연결해 확인)

### Step 6. 메뉴 루프로 연결 (`Main.java`)

**목표**: 메뉴를 반복 출력하고, 입력에 따라 자판기 기능을 호출한다.

**할 일**

1. `VendingMachine` 객체 생성.
2. `while`로 메뉴 반복.
3. 1~4 → `buy()`, 5 → 돈 넣기, 6 → 잔돈 반환 후 종료.

**힌트**

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        VendingMachine machine = new VendingMachine();

        while (true) {
            machine.printMenu();
            System.out.print("원하는 메뉴를 선택하세요 > ");
            int choice = sc.nextInt();

            if (choice >= 1 && choice <= 4) {
                machine.buy(choice);
            } else if (choice == 5) {
                System.out.print("넣을 금액 > ");
                machine.insertMoney(sc.nextInt());
            } else if (choice == 6) {
                System.out.println("\n잔돈 " + machine.returnChange() + "원이 반환되었습니다.");
                return;
            } else {
                System.out.println("잘못 입력하셨습니다. 다시 입력해주세요.");
            }
        }
    }
}
```

**확인**: 돈 넣고 음료 사고, 잔돈 부족도 처리되고, 종료 시 잔돈이 반환되면 성공!

### Step 7. 마무리 점검 (완성!)

**점검 항목**

- [ ] `Drink`는 `abstract`이고 직접 `new` 할 수 없는가?
- [ ] 네 음료가 모두 `dispense()`를 **다르게** 구현했는가?
- [ ] `buy()` 하나로 네 음료가 모두 처리되는가? (다형성 확인)
- [ ] 돈이 부족할 때 "잔돈이 부족합니다."가 나오는가?
- [ ] 종료 시 남은 금액이 잔돈으로 반환되는가?
- [ ] `totalMoney`가 `VendingMachine` 밖에서 직접 건드려지지 않는가? (캡슐화)

여기까지 통과하면 **OOP 자판기 완성입니다!** 🎉

## 6. 객체지향 학습 체크

- [ ] `abstract` 클래스와 `abstract` 메서드 작성
- [ ] `extends` + `super(...)`로 상속
- [ ] `@Override`로 메서드 재정의
- [ ] 부모 타입 배열(`Drink[]`)에 자식 객체 담기 (다형성)
- [ ] `drink.dispense()` 한 줄로 음료별 동작 실행
- [ ] 상태·로직을 클래스 안에 캡슐화

## 8. (선택) 도전 과제

1. **인터페이스 버전**: `Drink`를 `abstract class` 대신 `interface`로 바꿔 보고 차이 느끼기 (인터페이스는 필드·생성자를 못 가져서 각 음료가 이름·가격을 직접 가짐)
2. **재고**: 음료마다 남은 수량을 두고, 0이면 "품절입니다" 처리
3. **거스름돈**: 정확한 금액만 받게 하거나, 동전 단위로 거스름돈 계산
4. **음료 추가**: 커피 등 새 음료를 클래스 하나 추가 + 배열에 한 줄 추가로 끝나는지 확인 (절차지향이었다면 `switch`까지 고쳐야 했음 — OOP의 장점 체감!)
