# 세마포어 게임 만들기 (동시 입장 제한 던전)

> 세마포어로 **"정원이 정해진 던전"** 게임을 만듭니다.
> 모험가(스레드)들이 던전 입장을 두고 경쟁하는데, **동시에 N명까지만** 들어갈 수 있어요.
> 온라인 게임의 "인스턴스 던전 정원 / 대기열"이 바로 이 세마포어 원리예요.
> **아래 Step을 순서대로 따라가면, 정원 제한 던전 게임이 완성됩니다.**
>
> 💡 각 Step의 **힌트는 접혀 있어요.** 먼저 스스로 해보고, 막히면 "힌트 보기"를 펼치세요.

---

## 0. 먼저 알아둘 점

이 과제는 **세마포어(`Semaphore`)** 를 배웠다는 전제예요. (안 배웠다면: 세마포어는 "허가증 N장"을 들고 있다가 들어올 때 한 장씩 나눠주고, 없으면 기다리게 하는 도구예요.)
스레드 결과는 **실행마다 순서가 달라요**(비결정적). 우리는 정확한 순서가 아니라 **"동시에 던전 안에 최대 N명만 있는가"** 를 관찰하는 게 목표입니다.

---

## 1. 무엇을 만드나요?

정원이 **2명**인 던전에 모험가 5명이 입장을 시도합니다. 자리가 차면 나머지는 줄을 서서 기다려요.

```
전사 던전 입장 대기...
마법사 던전 입장 대기...
[입장] 전사 (남은 자리: 1/2)
[입장] 마법사 (남은 자리: 0/2)
궁수 던전 입장 대기...        ← 자리가 없어 대기
[클리어] 전사 → 320 골드 획득
[퇴장] 전사
[입장] 궁수 (남은 자리: 0/2)  ← 전사가 나가자 입장
...
```

---

## 2. 학습 목표

| 개념 | 어디서 배우나 |
|------|------|
| 세마포어 = 정원 | `Semaphore(N)` 으로 동시 입장 인원 제한 |
| acquire / release | 입장(자리 차지) / 퇴장(자리 반납) |
| availablePermits | 남은 자리 확인 |
| try-finally | 퇴장(release)을 반드시 보장 |
| 게임 설계 응용 | 인스턴스 던전 정원·대기열의 원리 |

---

## 3. 핵심 개념

### (1) 세마포어 = 던전 정원
세마포어는 **허가증(permit)을 N장** 들고 있어요. 던전 정원이 2명이면 허가증 2장.
- `acquire()` — 허가증 1장 받기 = **입장**(빈자리 없으면 대기)
- `release()` — 허가증 반납 = **퇴장**(자리 비움)

```java
Semaphore slots = new Semaphore(2);   // 정원 2명
slots.acquire();   // 입장 (자리 1개 차지)
// ... 던전 탐험 ...
slots.release();   // 퇴장 (자리 반납)
```

### (2) 왜 세마포어인가? (뮤텍스로는 안 됨)
뮤텍스는 "한 번에 1명"이라 정원 1짜리 던전밖에 못 만들어요. 우리는 **동시에 2명, 3명**처럼 N명을 받아야 하니 **세마포어**가 딱입니다.

### (3) 게임에서의 의미
온라인 게임에서 인스턴스 던전이 "정원 4명", 인기 던전은 "대기열 발생"이죠? 그 정원과 대기열이 바로 세마포어예요. 자리가 차면 `acquire()`에서 자동으로 줄 서서 기다리고, 누가 나가면(`release()`) 다음 사람이 들어갑니다.

> 팁: 퇴장(`release()`)은 **반드시 `finally`** 에 두세요. 탐험 중 예외가 나도 자리를 반납해야, 그 자리가 영영 막히지 않아요.

## 4. 파일 구조

| 파일 | 역할 |
|------|------|
| `Dungeon` | 던전. `Semaphore`로 정원 관리 + 입장(enter) |

| `Main` | 던전 하나에 모험가 여럿을 풀어놓고 실행 |

## 5. Step by Step

각 Step에 **목표 / 할 일 / 힌트(접힘) / 관찰 포인트**가 있습니다.

---

### Step 1. 던전 골격 (`Dungeon`)

**목표**: 정원만큼의 허가증을 가진 던전을 만든다.

**할 일**
1. `Semaphore slots` + 정원 `capacity` 필드.
2. 생성자에서 `new Semaphore(capacity)`.

<details>
<summary>💡 힌트 보기</summary>

```java
import java.util.concurrent.Semaphore;

class Dungeon {
    private final Semaphore slots;   // 동시 입장 가능 인원(허가증)
    private final int capacity;

    public Dungeon(int capacity) {
        this.capacity = capacity;
        this.slots = new Semaphore(capacity);
    }
}
```

</details>

**확인**: 컴파일 에러가 없으면 OK.

---

### Step 2. 입장 — enter() (`Dungeon`) · 핵심

**목표**: 모험가가 입장(자리 차지)해 잠시 탐험하고 골드를 얻은 뒤 퇴장(자리 반납)한다.

**할 일**
1. 입장 대기 메시지 출력.
2. `slots.acquire()` 로 자리 차지 → 입장 메시지 + 남은 자리(`availablePermits()`).
3. 랜덤 시간 탐험(`Thread.sleep`) + 랜덤 골드.
4. **`finally`에서 `slots.release()`** 로 퇴장.

<details>
<summary>💡 힌트 보기</summary>

```java
public void enter(String name) throws InterruptedException {
    System.out.println(name + " 던전 입장 대기...");
    slots.acquire();                 // 빈자리 1개 차지 (없으면 대기)
    try {
        System.out.println("[입장] " + name
            + " (남은 자리: " + slots.availablePermits() + "/" + capacity + ")");

        Thread.sleep((int)(Math.random() * 2000) + 1000);  // 1~3초 탐험
        int gold = (int)(Math.random() * 400) + 100;       // 100~500 골드
        System.out.println("[클리어] " + name + " → " + gold + " 골드 획득");
    } finally {
        System.out.println("[퇴장] " + name);
        slots.release();             // ★ 자리 반납 (예외가 나도 반드시!)
    }
}
```

`release()`를 `finally`에 두지 않으면, 탐험 중 사고가 났을 때 그 자리가 영영 막혀버려요.

</details>

**관찰 포인트**: (실행은 Step 4) 입장 시 "남은 자리"가 줄고, 퇴장 시 다음 모험가가 들어오는 구조면 OK.

---

### Step 3. 모험가 스레드 (`Adventurer`)

**목표**: 모험가가 던전 입장을 시도하는 스레드를 만든다.

<details>
<summary>💡 힌트 보기</summary>

```java
class Adventurer extends Thread {
    private Dungeon dungeon;
    private String name;

    public Adventurer(Dungeon dungeon, String name) {
        this.dungeon = dungeon;
        this.name = name;
    }

    public void run() {
        try {
            dungeon.enter(name);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
    }
}
```

</details>

**확인**: 같은 `dungeon`을 받아 입장을 시도하는 구조면 OK.

---

### Step 4. main 실행 — 정원 2, 모험가 5명

**목표**: **하나의 던전**(정원 2)에 모험가 5명을 풀어놓고 실행한다.

<details>
<summary>💡 힌트 보기</summary>

```java
public static void main(String[] args) {
    Dungeon dungeon = new Dungeon(2);   // 정원 2명짜리 던전 하나!

    String[] names = { "전사", "마법사", "궁수", "도적", "성기사" };
    for (String name : names) {
        new Adventurer(dungeon, name).start();
    }
}
```

**중요**: 던전을 **하나만** 만들어 모두가 공유해야 정원 경쟁이 일어나요. 각자 던전을 만들면 의미가 없어요.

</details>

**관찰 포인트**: 5명이 입장하려 해도 **던전 안엔 동시에 최대 2명**만 있어요. 나머지는 "입장 대기..." 상태로 기다리다가, 누가 `[퇴장]`하면 그 자리로 들어갑니다. "남은 자리"가 0을 넘어가지 않는지 확인!

## 8. (선택) 도전 과제

1. **정원 바꾸기**: 정원을 1, 3으로 바꿔 동작이 어떻게 달라지는지 (정원 1 = 뮤텍스와 같아짐!)
2. **골드 랭킹(스레드 안전)**: 각 모험가가 번 골드를 합산해 누가 1등인지 — 공유 점수는 `AtomicInteger`나 `synchronized`로 안전하게 (스레드가 끝나길 기다리려면 `join`)
3. **몬스터 이벤트**: 탐험 중 랜덤으로 "보스 조우!"가 뜨면 더 오래 걸리고 골드 2배 같은 이벤트 추가
4. **반복 도전**: 모험가가 던전을 1번이 아니라 3번 반복 입장(while/for)하게 해서 더 치열한 경쟁 만들기
5. **공정 입장(줄서기)**: `new Semaphore(2, true)` 공정 모드로, 먼저 기다린 모험가가 먼저 들어가게 하기
6. **tryAcquire**: "5초 안에 자리 안 나면 포기하고 마을로 돌아감"을 `tryAcquire(5, TimeUnit.SECONDS)`로 구현
