# 회원 관리 프로그램 만들기 (2차원 배열)

> `String[][]` 2차원 배열에 회원 정보를 저장하고, 추가·조회·수정·삭제(CRUD)하는 콘솔 프로그램을 만듭니다. 배열, static 변수/메서드, 선형 검색, 배열 당기기를 배웁니다. **아래 Step을 순서대로 따라가면, 마지막 Step에서 프로그램이 완성됩니다.**

---

## 1. 무엇을 만드나요?

시작하면 요금제를 고르고(정원이 정해짐), 메뉴가 반복해서 나옵니다.

```
[요금제를 선택하세요]
[1]Lite : 10명 [2]Basic : 20명 [3]Premium : 30명
> 2

[수행할 업무를 선택하세요 - 현재 회원수 : 0/20]
[1]회원추가 [2]회원조회(메일) [3]회원조회(이름)
[4]회원전체조회 [5]회원정보 수정 [6]회원삭제
[7]프로그램 종료
>
```

---

## 2. 요구사항 정리 (기능 명세)

|번호|기능|설명|
|---|---|---|
|0|요금제 선택|Lite(10) / Basic(20) / Premium(30) 중 선택 → 그만큼 **정원**이 정해진다|
|1|회원 추가|이름·이메일·연락처를 입력받아 저장. **정원 초과**·**이메일 중복**은 막는다|
|2|회원 조회(메일)|이메일로 찾아 한 명의 정보를 보여준다|
|3|회원 조회(이름)|이름으로 찾아 정보를 보여준다|
|4|회원 전체 조회|등록된 모든 회원을 출력한다|
|5|회원 정보 수정|이메일로 찾아 이름·이메일·연락처를 새 값으로 바꾼다|
|6|회원 삭제|이메일로 찾아 삭제하고, 뒤 회원들을 **앞으로 당긴다**|
|7|종료|프로그램을 끝낸다|

> **이메일이 "고유 식별자"** 입니다. 조회·수정·삭제는 모두 이메일을 기준으로 한 명을 찾습니다.

## 3. 핵심 개념

### (1) 2차원 배열로 회원 저장

회원 한 명 = 한 줄(행), 속성 3개 = 칸(열)으로 표현합니다.

```
String[][] members = new String[정원][3];

         열0(이름)   열1(이메일)        열2(연락처)
행0 →   "홍길동"    "hong@a.com"      "010-1111"
행1 →   "김철수"    "kim@b.com"       "010-2222"
 ...
```

- `members[i][0]` = i번째 회원 이름
- `members[i][1]` = i번째 회원 이메일
- `members[i][2]` = i번째 회원 연락처

### (2) static 변수로 상태 공유

여러 메서드가 함께 쓰는 값은 클래스 차원의 `static` 변수로 둡니다.

```java
static int totalCnt = 0;   // 정원 (요금제로 결정)
static int memberCnt = 0;  // 현재 회원 수 (= 다음에 채울 칸의 인덱스)
```

`memberCnt`는 "현재 인원 수"이자 "다음에 회원을 넣을 빈 칸의 위치"라는 두 의미를 함께 가집니다.

### (3) 선형 검색 패턴

이메일/이름으로 찾을 때는 처음부터 끝까지 돌면서 `equals`로 비교합니다.

```java
for (int i = 0; i < members.length; i++) {
    if ( 찾는값.equals(members[i][열]) ) {
        // 찾음! 처리 후 return
    }
}
// 여기까지 왔다면 못 찾은 것
```

> 문자열 비교는 항상 `.equals()`로! `==`는 쓰지 않습니다.

### (4) 삭제 후 "당기기"

배열은 중간을 비울 수 없으므로, 삭제한 칸 **뒤의 회원들을 한 칸씩 앞으로** 옮깁니다.

```
삭제 전: [A][B][C][D]      B를 삭제
당기기:  [A][C][D][ ]      C,D가 한 칸씩 앞으로
```

## 4. 프로그램 구조 (메서드 역할)

|메서드|역할|
|---|---|
|`printPricePlan()`|요금제 메뉴 출력, 선택 번호 반환|
|`printMenu(memberCnt)`|업무 메뉴 출력(현재인원/정원 포함), 선택 번호 반환|
|`addMember(members)`|회원 추가 (정원·중복 검사 포함)|
|`checkEmail(members, email)`|이메일 중복 여부(true/false) 판단|
|`selectEmail(members)`|이메일로 조회|
|`selectName(members)`|이름으로 조회|
|`selectAll(members)`|전체 조회|
|`updateMember(members)`|정보 수정|
|`deleteMember(members)`|삭제 + 당기기|
|`main()`|전체 흐름: 요금제 → 배열 생성 → 메뉴 반복|

## 5. Step by Step

각 Step에 **목표 / 할 일 / 힌트 / 확인 방법**이 있습니다.

---

### Step 1. 요금제 선택 + 메뉴 골격 (`printPricePlan`, `printMenu`, `main`)

**목표**: 요금제를 고르면 그만큼 배열을 만들고, 메뉴가 반복 출력되며 7번이면 종료된다.

**할 일**

1. `printPricePlan()`: 요금제 안내 출력 후 선택 번호(1~3) 반환.
2. `main()`: 선택값 `num`으로 `new String[num*10][3]` 배열 생성, `totalCnt = num*10`.
3. `printMenu(memberCnt)`: 현재인원/정원과 업무 메뉴 출력, 선택 번호 반환.
4. `while(true)` + `switch`: 각 case는 빈 채로 두고, case 7에서 `return`, default는 안내.

**힌트**

```java
static int totalCnt = 0;
static int memberCnt = 0;

public static int printPricePlan() {
    Scanner sc = new Scanner(System.in);
    System.out.println("[요금제를 선택하세요]");
    System.out.println("[1]Lite : 10명 [2]Basic : 20명 [3]Premium : 30명");
    return sc.nextInt();
}
// main()
int num = printPricePlan();
String[][] members = new String[num * 10][3];
totalCnt = num * 10;
while (true) {
    int choice = printMenu(memberCnt);
    switch (choice) {
        case 1: /* Step2 */ break;
        // ...
        case 7: System.out.println("이용해주셔서 감사합니다."); return;
        default: System.out.println("올바른 번호를 입력하세요.");
    }
}
```

**확인**: 요금제 선택 후 메뉴가 반복되고, 7을 누르면 종료되면 성공.

---

### Step 2. 회원 추가 + 중복 체크 (`addMember`, `checkEmail`)

**목표**: 이름·이메일·연락처를 받아 배열에 저장한다. 정원 초과와 이메일 중복은 막는다.

**할 일**

1. `memberCnt == members.length`이면 "회원이 꽉 찼습니다." 출력 후 종료.
2. 이름·이메일·연락처를 입력받는다.
3. `checkEmail(members, email)`이 `true`면 "이미 존재하는 회원입니다." 출력 후 종료.
4. `members[memberCnt]`에 세 값을 채우고 `memberCnt++`.

**힌트**

```java
public static boolean checkEmail(String[][] members, String email) {
    for (int i = 0; i < members.length; i++) {
        if (email.equals(members[i][1])) return true; // 이미 있음
    }
    return false;
}
// addMember 안
members[memberCnt][0] = name;
members[memberCnt][1] = email;
members[memberCnt][2] = phone;
memberCnt++;
```

> ⚠️ **Scanner 주의 (꼭 읽기)**: 메뉴 번호는 `nextInt()`로 받고 이름 등은 `nextLine()`으로 받는데, `nextInt()`는 줄바꿈(Enter)을 버퍼에 남깁니다. 그러면 다음 `nextLine()`이 **빈 문자열**을 읽어 이름이 비는 버그가 생깁니다. 해결책은 Step 8 참고 — **Scanner를 하나만 만들어 공유하고, `nextInt()` 뒤에 `nextLine()`으로 남은 줄바꿈을 한 번 비워주는 것**입니다.

**확인**: 추가 후 전체 조회(Step 5)에서 보이고, 같은 이메일로 또 추가하면 거부되면 성공.

---

### Step 3. 이메일로 조회 (`selectEmail`)

**목표**: 입력한 이메일과 같은 회원을 찾아 출력한다.

**할 일**

1. 이메일 입력받기.
2. 선형 검색으로 `members[i][1]`과 비교.
3. 찾으면 정보 출력 후 `return`, 끝까지 못 찾으면 "찾으시는 정보가 없습니다.".

**힌트**

```java
for (int i = 0; i < members.length; i++) {
    if (email.equals(members[i][1])) {
        System.out.println("[이름] " + members[i][0] +
                           ", [이메일] " + members[i][1] +
                           ", [연락처] " + members[i][2]);
        return;
    }
}
System.out.println("찾으시는 정보가 없습니다.");
```

**확인**: 등록한 이메일로 조회하면 정보가 나오고, 없는 이메일은 안내 메시지가 나오면 성공.

---

### Step 4. 이름으로 조회 (`selectName`)

**목표**: Step 3과 똑같되, 비교 대상을 이메일(`[1]`)이 아니라 **이름(`[0]`)** 으로 바꾼다.

**할 일**: `selectEmail`을 복사해 비교 열만 `members[i][0]`으로 바꾼다.

**확인**: 이름으로 조회가 되면 성공.

> 같은 이름이 여러 명이면 어떻게 될까요? 지금은 **처음 찾은 한 명만** 나옵니다. (도전 과제에서 개선)

---

### Step 5. 전체 조회 (`selectAll`)

**목표**: 등록된 회원을 모두 출력한다.

**할 일**: 반복문으로 모든 회원 정보를 출력.

**힌트**

```java
for (int i = 0; i < memberCnt; i++) {   // memberCnt 까지만! (빈 칸 null 출력 방지)
    System.out.println("[이름] " + members[i][0] +
                       ", [이메일] " + members[i][1] +
                       ", [연락처] " + members[i][2]);
}
```

> **팁**: `members.length`(정원)까지 돌면 아직 안 채운 칸이 `null`로 출력됩니다. 등록된 인원인 `memberCnt`까지만 도는 것이 깔끔합니다.

**확인**: 등록한 회원만 빠짐없이, 빈 칸 없이 나오면 성공.

---

### Step 6. 정보 수정 (`updateMember`)

**목표**: 이메일로 회원을 찾아 이름·이메일·연락처를 새 값으로 바꾼다.

**할 일**

1. 이메일을 입력받아 선형 검색으로 인덱스 `idx`를 찾는다.
2. 못 찾으면(`idx == -1`) "찾으시는 회원이 없습니다." 후 종료.
3. 새 이름·이메일·연락처를 입력받아 `members[idx]`를 덮어쓴다.
4. "수정이 완료되었습니다." 출력.

**힌트 (인덱스 찾기 패턴)**

```java
int idx = -1;
for (int i = 0; i < members.length; i++) {
    if (email.equals(members[i][1])) { idx = i; break; }
}
if (idx == -1) { System.out.println("찾으시는 회원이 없습니다."); return; }
// 이후 members[idx][0/1/2] 새 값으로 교체
```

**확인**: 수정 후 조회하면 바뀐 정보가 나오면 성공.

---

### Step 7. 회원 삭제 + 당기기 (`deleteMember`)

**목표**: 이메일로 찾아 삭제하고, 뒤 회원들을 한 칸씩 앞으로 당겨 빈자리를 없앤다.

**할 일**

1. 이메일로 `idx`를 찾는다. 없으면 종료.
2. `memberCnt--`.
3. `idx`부터 끝-1까지, 다음 회원을 현재 자리로 복사해 **당긴다**.
4. (권장) 당긴 뒤 마지막에 남는 중복 데이터를 `null`로 정리한다.

**힌트 (당기기 — 이 Step의 핵심)**

```java
for (int i = idx; i < members.length - 1; i++) {
    members[i][0] = members[i + 1][0];
    members[i][1] = members[i + 1][1];
    members[i][2] = members[i + 1][2];
}
// 권장: 마지막 칸 정리
members[memberCnt][0] = null;
members[memberCnt][1] = null;
members[memberCnt][2] = null;
```

**확인**: 가운데 회원을 지워도 목록이 빈칸 없이 이어지고, 현재 인원수가 줄면 성공.

---

### Step 8. 마무리 다듬기 (완성!)

**점검 항목**

- [ ] **Scanner 문제 해결**: 입력이 한 칸씩 밀리거나 이름이 비는가? → 아래 권장 방식 적용.
- [ ] 현재 인원/정원(`memberCnt`/`totalCnt`)이 메뉴에 정확히 표시되는가?
- [ ] 정원이 꽉 찼을 때 추가가 막히는가?
- [ ] 삭제 후 전체 조회 시 빈칸이나 중복 회원이 안 나오는가?
- [ ] 없는 회원을 조회/수정/삭제해도 프로그램이 멈추지 않는가?

**Scanner 권장 방식**

```java
static Scanner sc = new Scanner(System.in);  // 하나만 만들어 모든 메서드가 공유

// nextInt() 뒤에는 남은 줄바꿈을 비워준다
int num = sc.nextInt();
sc.nextLine();   // ★ 버퍼에 남은 Enter 제거
```

여기까지 통과하면 **회원 관리 프로그램 완성입니다!** 🎉

## 7. 개선 & 도전 과제

**먼저 다듬으면 좋은 점**

1. 전체 조회를 `memberCnt`까지만 돌기 (빈 칸 `null` 출력 방지)
2. 삭제 후 마지막 칸 `null` 정리 (당기고 남은 꼬리 데이터 제거)
3. `Scanner`를 하나만 만들어 공유 (다중 Scanner는 입력 꼬임의 원인)
4. 표준 실행을 위해 `public static void main(String[] args)` 형태로 두기

**확장 도전 과제**

1. **회원을 클래스로**: `Member` 클래스(이름/이메일/연락처)를 만들어 `Member[]`로 관리 (2차원 배열보다 깔끔)
2. **ArrayList로**: 고정 크기 배열 대신 `ArrayList<Member>`로 바꿔 정원 제한·당기기 코드 제거
3. **동명이인 처리**: 이름 조회 시 일치하는 모두 출력
4. **파일 저장**: 프로그램을 꺼도 회원이 유지되도록 파일로 저장/불러오기
5. **입력 검증**: 이메일 형식·연락처 숫자 형식 검사
