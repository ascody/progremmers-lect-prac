# 자바 람다 기초부터 익히기 (익명 클래스를 한 줄로 줄이기)

> 람다(lambda)를, **익명 클래스에서 출발해 한 줄로 줄여가며** 익혀요. 똑같이 동작하는 코드를 점점 짧게 만드는 과정에서 문법이 자연스럽게 손에 붙어요.
> 마지막엔 람다가 가장 빛나는 **리스트 정렬(`Comparator`)** 까지 해봐요.
> **아래 Step을 순서대로 따라가면, 마지막 Step에서 람다를 자유롭게 쓰는 프로그램이 완성됩니다.**
>
> 💡 각 Step의 **힌트는 접혀 있어요.** "이걸 어떻게 더 줄이지?"를 먼저 고민하고, 막히면 펼치세요.

---

## 0. 먼저 알아둘 점

- 람다는 **함수형 인터페이스**(추상 메서드가 딱 하나인 인터페이스)가 필요한 자리에만 쓸 수 있어요. 그래서 이 가이드는 **간단한 함수형 인터페이스를 직접 만드는 것**부터 시작해요.
- 람다는 새로운 마법이 아니라, **익명 클래스(이름 없는 클래스)를 짧게 줄인 표현**이에요. 그래서 익명 클래스 모양을 먼저 보고, 거기서 군더더기를 하나씩 지워가요.
- 인터페이스, 익명 클래스, `ArrayList<String>` 같은 컬렉션 사용은 안다고 가정해요.
- 직접 만드는 `<T>` 제네릭은 다루지 않아요. (`Comparator<String>`처럼 **이미 만들어진 걸 구체 타입으로 쓰는 것**만 해요.)

---

## 1. 무엇을 만드나요?

익명 클래스를 람다로 줄이고, 매개변수 개수별로 람다를 쓰고, 리스트를 정렬하는 프로그램을 완성해요. 다 만들면 이렇게 나와요.

```
===== 1. 익명 클래스 vs 람다 (같은 동작) =====
익명 클래스 add: 7
람다 add: 7

===== 2. 람다로 만든 연산들 =====
3 + 4 = 7
9 - 2 = 7
3 * 5 = 15

===== 3. 매개변수 개수별 람다 =====
(0개) 안녕하세요, 람다!
(1개) [로그] 시작합니다
(2개) 10 + 20 = 30

===== 4. 실전: Comparator로 길이순 정렬 =====
정렬 전: [가나다, 가, 라마]
정렬 후: [가, 라마, 가나다]
```

핵심은 1번이에요. **익명 클래스로 쓴 add와 람다로 쓴 add가 완전히 같은 일**을 해요. 람다는 그걸 짧게 쓴 것뿐이에요.

## 3. 핵심 개념

### (1) 함수형 인터페이스 — 람다가 들어갈 '틀'
**추상 메서드가 딱 하나**인 인터페이스를 함수형 인터페이스라고 해요. 람다는 그 하나뿐인 메서드의 **본문**을 짧게 적는 거예요.
```java
@FunctionalInterface              // 추상 메서드가 1개임을 표시(안 붙여도 되지만 권장)
interface Operation {
    int apply(int a, int b);      // 이 하나뿐인 메서드를 람다로 채운다
}
```

### (2) 익명 클래스 → 람다 (군더더기 지우기)
같은 동작을 두 방식으로 적으면 이렇게 달라요.
```java
// 익명 클래스: 길다
Operation add = new Operation() {
    @Override
    public int apply(int a, int b) { return a + b; }
};

// 람다: 핵심만 (매개변수 -> 결과)
Operation add = (a, b) -> a + b;
```
`new Operation()`, 메서드 이름 `apply`, 타입, `return`, 중괄호… 이 **반복되는 껍데기를 다 지우고** `(a, b) -> a + b`만 남긴 게 람다예요. "**무엇을** 받아서(`a, b`) **무엇을** 돌려주는지(`a + b`)"만 적어요.

### (3) 람다 문법 축약 3단계
```
(1) 완전한 형태   (int a, int b) -> { return a + b; }
(2) 타입 생략     (a, b) -> { return a + b; }        ← 타입은 인터페이스가 알려줌
(3) 한 줄 식      (a, b) -> a + b                    ← 중괄호·return·세미콜론 생략
```
한 줄짜리 식이면 `{ }`와 `return`을 생략해요. 본문이 여러 줄이면 `{ }`를 쓰고 `return`도 직접 써요.

### (4) 매개변수 개수별 모양
```
0개:  () -> System.out.println("hi")     // 괄호는 비워둠(생략 불가)
1개:  msg -> System.out.println(msg)     // 괄호 생략 가능
2개:  (a, b) -> a + b                     // 괄호 필요
```

```
[기억법] 람다 = "익명 클래스에서 껍데기 빼고 (매개변수) -> (결과) 만 남기기"
```

## 4. 파일 구조

|------|------|
| `Operation.java` | 함수형 인터페이스 — `int apply(int a, int b)` |

## 5. Step by Step

### Step 1. 함수형 인터페이스 + 익명 클래스 (출발점) (`Operation.java`, `Main.java`)

**목표**: 람다로 줄이기 *전*의 모습(익명 클래스)을 먼저 만들어요.

**할 일**
1. `Operation` 인터페이스를 만들어요. 메서드는 `int apply(int a, int b);`
2. `main`에서 **익명 클래스**로 "더하기"를 구현해 `add`에 담고, `add.apply(3, 4)`를 출력해요.

<details>
<summary>💡 힌트 보기</summary>

```java
// Operation.java
@FunctionalInterface
interface Operation {
    int apply(int a, int b);
}
```
```java
// Main.java 안에서
Operation add = new Operation() {
    @Override
    public int apply(int a, int b) {
        return a + b;
    }
};
System.out.println(add.apply(3, 4));   // 7
```

`@FunctionalInterface`는 "추상 메서드가 하나뿐"임을 컴파일러가 검사해줘요. 실수로 두 개 만들면 에러로 알려줘요.

</details>

**확인**: 익명 클래스로 만든 `add`가 `7`을 출력하면 통과예요. 이걸 Step 2에서 람다로 줄여요.

---

### Step 2. 익명 클래스를 람다로 바꾸기 (`Main.java`)

**목표**: Step 1의 익명 클래스를 **완전히 같은 동작의 람다**로 바꿔요.

**할 일**
1. Step 1의 익명 클래스 `add`를 람다 `(a, b) -> a + b`로 다시 써요.
2. 익명 클래스 버전과 람다 버전이 **같은 결과(7)** 를 내는지 둘 다 출력해 비교해요.

<details>
<summary>💡 힌트 보기</summary>

```java
Operation addAnon   = new Operation() {
    @Override public int apply(int a, int b) { return a + b; }
};
Operation addLambda = (a, b) -> a + b;     // 위와 똑같은 일을 한다

System.out.println("익명 클래스 add: " + addAnon.apply(3, 4));   // 7
System.out.println("람다 add: "      + addLambda.apply(3, 4));   // 7
```

`new Operation() { ... apply ... }` 전체가 `(a, b) -> a + b` 한 조각으로 줄었어요. 받는 타입은 그대로 `Operation`이에요.

</details>

**확인**: 두 버전 모두 `7`을 출력하면 성공이에요. "껍데기만 줄었을 뿐 같은 것"임을 확인하는 게 핵심이에요.

---

### Step 3. 람다 축약 3단계 익히기 (`Main.java`)

**목표**: 람다를 **완전한 형태 → 한 줄 식**까지 줄이는 과정을 직접 써봐요.

**할 일**
1. 같은 더하기 람다를 3가지 형태로 적어보고, 셋 다 똑같이 동작함을 확인해요.
2. 그 감으로 빼기·곱하기 람다(`sub`, `mul`)를 **한 줄 식**으로 만들어요.

<details>
<summary>💡 힌트 보기</summary>

```java
Operation add1 = (int a, int b) -> { return a + b; };  // (1) 완전한 형태
Operation add2 = (a, b) -> { return a + b; };          // (2) 타입 생략
Operation add3 = (a, b) -> a + b;                       // (3) 한 줄 식

Operation sub = (a, b) -> a - b;
Operation mul = (a, b) -> a * b;
System.out.println(sub.apply(9, 2));   // 7
System.out.println(mul.apply(3, 5));   // 15
```

본문이 **한 줄(식)** 이면 `{ }`·`return`·`;`를 다 생략해요. 여러 줄이면 `{ }`를 쓰고 `return`도 직접 적어야 해요.

</details>

**확인**: 세 형태가 같은 값을 내고, `sub`/`mul`이 `7`/`15`를 출력하면 성공이에요.

---

### Step 4. 매개변수 개수별 람다 (`Printer.java`, `Main.java`)

**목표**: 매개변수가 **0개·1개·2개**일 때 람다 모양이 어떻게 달라지는지 익혀요.

**할 일**
1. **0개**: `Runnable`(자바 기본 제공)을 람다로 만들어 인사를 출력해요. → `() -> ...`
2. **1개**: `Printer` 인터페이스(`void print(String msg)`)를 만들고, 람다로 로그를 찍어요. → `msg -> ...`
3. **2개**: 이미 만든 `Operation` 더하기를 호출해요. → `(a, b) -> ...`

<details>
<summary>💡 힌트 보기</summary>

```java
// Printer.java
@FunctionalInterface
interface Printer { void print(String msg); }
```
```java
// Main.java 안에서
Runnable hello = () -> System.out.println("(0개) 안녕하세요, 람다!");
hello.run();

Printer log = msg -> System.out.println("(1개) [로그] " + msg);  // 괄호 생략
log.print("시작합니다");

Operation add = (a, b) -> a + b;
System.out.println("(2개) 10 + 20 = " + add.apply(10, 20));
```

매개변수가 **0개면 빈 괄호 `()`** 가 꼭 필요하고, **1개면 괄호를 생략**할 수 있고(`msg -> ...`), **2개 이상이면 괄호가 필요**해요(`(a, b) -> ...`).

</details>

**확인**: 0개·1개·2개 람다가 각각 잘 출력되면 성공이에요.

---

### Step 5. 실전 — Comparator로 리스트 정렬 (`Main.java`)

**목표**: 람다가 실제로 빛나는 자리 — **정렬 기준을 람다로** 넘겨봐요.

**할 일**
1. 문자열 `ArrayList`를 만들어요. (예: `"가나다"`, `"가"`, `"라마"`)
2. `list.sort(...)`에 **두 값을 비교하는 람다**를 넘겨 **글자 수 오름차순**으로 정렬해요.
3. 정렬 전후를 출력해요.

<details>
<summary>💡 힌트 보기</summary>

```java
import java.util.ArrayList;
import java.util.Arrays;

ArrayList<String> names = new ArrayList<>(Arrays.asList("가나다", "가", "라마"));
System.out.println("정렬 전: " + names);

names.sort((s1, s2) -> s1.length() - s2.length());   // 길이 차이로 비교

System.out.println("정렬 후: " + names);   // [가, 라마, 가나다]
```

`sort`는 `Comparator`(두 값을 비교해 음수/0/양수를 돌려주는 함수형 인터페이스)를 받아요. `(s1, s2) -> s1.length() - s2.length()`가 "앞 글자 수 - 뒤 글자 수"라, 음수면 앞이 먼저 와서 **짧은 것부터** 정렬돼요.

</details>

**확인**: 정렬 후가 `[가, 라마, 가나다]`(짧은 것부터)면 성공이에요.

## 6. 학습 체크

- [ ] 함수형 인터페이스가 무엇인지(추상 메서드 1개) 말할 수 있다
- [ ] 익명 클래스와 람다가 같은 일을 한다는 걸 설명할 수 있다
- [ ] 람다에서 타입·중괄호·`return`을 언제 생략하는지 안다
- [ ] 매개변수 0개/1개/2개의 괄호 규칙을 안다
- [ ] `sort`에 비교 람다를 넘겨 정렬할 수 있다

## 7. 최종 완성 체크리스트

- [ ] `Operation`, `Printer`가 함수형 인터페이스로 선언됐다(`@FunctionalInterface`)
- [ ] 익명 클래스 → 람다 변환을 직접 해봤다
- [ ] 한 줄 식 람다(`(a,b) -> a+b`)를 쓸 수 있다
- [ ] `Runnable`(0개), `Printer`(1개) 람다를 만들었다
- [ ] `Comparator` 람다로 정렬이 된다

## 8. (선택) 도전 과제

1. `Operation`으로 나누기 람다 `(a, b) -> a / b`를 만들고, `0`으로 나눌 때 어떻게 되는지 확인해보세요.
2. `Comparator` 람다를 바꿔 **긴 글자부터(내림차순)** 정렬해보세요. (`s2.length() - s1.length()`)
3. 정렬 기준을 "글자 수"가 아니라 "**가나다 사전순**"으로 바꿔보세요. (`s1.compareTo(s2)`)
4. ★ 자바가 기본 제공하는 함수형 인터페이스 `Predicate<String>`(조건 검사: `test`)을 람다로 만들어, 문자열이 비었는지 검사해보세요. (`s -> s.isEmpty()`)
5. ★ **메서드 참조**(`::`)를 조사해보세요. `s -> System.out.println(s)`는 `System.out::println`으로, `(s1, s2) -> s1.length() - ...` 같은 일부 람다는 더 짧게 쓸 수 있어요.
